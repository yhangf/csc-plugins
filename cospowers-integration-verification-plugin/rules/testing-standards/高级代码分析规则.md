# 高级代码静态分析规则

## 用途

本文档供 AI 执行两类任务：**代码生成**（写代码前对照质量基线）、**代码 Review**（逐条扫描输出违规）。

违规报告格式：`[规则ID] <文件>:<行号> — <描述>`

强制等级：`[强制]` 必须遵守 / `[建议]` 推荐遵守

---

## 分析工具速查

| 语言 | 工具 | 检查项 |
|------|------|--------|
| Python | `flake8` | 代码风格、基础错误 |
| Python | `pylint` | 复杂度、可维护性、重复代码 |
| Python | `bandit` | 安全漏洞 |
| Python | `radon` | 圈复杂度（CC）、可维护性指数（MI） |
| Go | `go vet` | 静态错误检查 |
| Go | `golangci-lint` | 复杂度、安全、重复代码综合检查 |
| Go | `gocyclo` | 圈复杂度 |
| 通用 | `SonarQube` | 技术债务、覆盖率、重复率 |

## 规则详情

### 命名语义

**STATIC-NAME-01** `[强制]` 布尔类型变量和返回布尔值的函数必须使用 `is_`、`has_`、`can_`、`allow_`、`should_` 等语义前缀，清晰表达其判断语义。

违规信号：布尔变量命名为 `flag`、`status`、`valid`、`check`、`enable` 等无判断语义的词；返回 bool 的函数名为动词（`validate`、`check`）但未加语义前缀。

```python
# 变量
❌ flag = True
❌ status = False
❌ enable = user.active
❌ check = validate_age(age)

✅ is_active = True
✅ has_permission = False
✅ is_enabled = user.active
✅ is_valid_age = validate_age(age)

# 函数
❌ def check_user(user): ...        # 返回 bool 但前缀含糊
❌ def validate_email(email): ...   # validate 暗示可能抛异常而非返回 bool

✅ def is_valid_user(user) -> bool: ...
✅ def has_permission(user, action) -> bool: ...
✅ def can_access(user, resource) -> bool: ...
✅ def is_expired(expires_at) -> bool: ...
```

```go
// 变量
❌ flag := true
❌ valid := false

✅ isActive := true
✅ hasPermission := false

// 函数
❌ func CheckUser(user *User) bool { ... }

✅ func IsValidUser(user *User) bool { ... }
✅ func HasPermission(user *User, action string) bool { ... }
✅ func CanAccess(user *User, resource string) bool { ... }
```

**STATIC-NAME-02** `[强制]` 函数/方法名使用**动词+宾语**形式，禁止泛化命名（`do`、`handle`、`process`、`manage`、`deal`、`run` 等单独使用）。

违规信号：函数名为单个动词或泛化词，无法从名称判断操作了什么资源；多个不同功能的函数使用相同的泛化前缀导致混乱。

```python
# 泛化命名，无法判断操作对象
❌ def handle(data): ...
❌ def process(req): ...
❌ def do_something(): ...
❌ def manage_resource(): ...        # manage 什么？对 resource 做什么？
❌ def run(): ...

# 动词 + 宾语，清晰表达意图
✅ def create_vm(params): ...
✅ def delete_snapshot(snapshot_id): ...
✅ def list_expired_trusts(project_id): ...
✅ def send_alarm_notification(alarm): ...
✅ def calculate_quota_usage(project_id): ...
```

```go
// 泛化命名
❌ func Handle(ctx context.Context, data interface{}) error
❌ func Process(req *Request) (*Response, error)
❌ func Run(task Task) error

// 动词 + 宾语
✅ func CreateVM(ctx context.Context, params *VMParams) (*VM, error)
✅ func DeleteSnapshot(ctx context.Context, snapshotID string) error
✅ func ListExpiredTrusts(ctx context.Context, projectID string) ([]*Trust, error)
```

常见动词对照（选精确词替代泛化词）：

| 泛化词 | 精确替代 |
|-------|---------|
| `handle` | `process_alarm`、`dispatch_event`、`route_request` |
| `process` | `parse_config`、`transform_data`、`validate_params` |
| `do` | 直接描述动作：`send_email`、`create_record` |
| `manage` | `assign_quota`、`revoke_permission`、`schedule_task` |
| `run` | `execute_migration`、`trigger_backup`、`start_scanner` |

**STATIC-NAME-03** `[强制]` 常量使用全大写下划线命名（`UPPER_SNAKE_CASE`）；禁止在代码逻辑中直接使用魔法数字（Magic Number）和魔法字符串，必须提取为具名常量。

违规信号：代码中直接出现无命名含义的数字（`86400`、`1000`、`3`）或字符串（`"admin"`、`"running"`、`"v1"`）；常量命名使用小写或驼峰。

```python
# 常量命名错误
❌ maxConnections = 100       # 驼峰，应全大写
❌ max_connections = 100      # 下划线但非全大写

✅ MAX_CONNECTIONS = 100
✅ DEFAULT_TIMEOUT_SECONDS = 30
✅ VM_STATUS_RUNNING = "running"

# 魔法数字/字符串
❌ if age > 150:                           # 150 含义不明
       raise ValueError("invalid age")

❌ if user.role == "admin":                # "admin" 硬编码
       pass

❌ time.sleep(86400)                       # 86400 是什么？

❌ if len(items) > 1000:                   # 1000 是什么阈值？
       paginate()

✅ MAX_VALID_AGE = 150
   if age > MAX_VALID_AGE:
       raise ValueError("age exceeds limit")

✅ ROLE_ADMIN = "admin"
   if user.role == ROLE_ADMIN:
       pass

✅ SECONDS_PER_DAY = 86400
   time.sleep(SECONDS_PER_DAY)

✅ MAX_PAGE_SIZE = 1000
   if len(items) > MAX_PAGE_SIZE:
       paginate()
```

```go
// 魔法数字/字符串
❌ if age > 150 { return errors.New("invalid") }
❌ if user.Role == "admin" { ... }
❌ time.Sleep(86400 * time.Second)

✅ const (
       MaxValidAge      = 150
       RoleAdmin        = "admin"
       SecondsPerDay    = 86400
       MaxPageSize      = 1000
       VMStatusRunning  = "running"
   )

   if age > MaxValidAge { return errors.New("age exceeds limit") }
   if user.Role == RoleAdmin { ... }
   time.Sleep(SecondsPerDay * time.Second)
```

例外情况（允许直接使用的字面量）：
- `0`、`1`、`-1` 在明确上下文中（索引、布尔比较、错误码约定）
- 循环变量初始值 `i := 0`
- 测试代码中的期望值

违规信号：新增的 `.py`/`.go` 文件无文件头说明；docstring 仅有文件名无实质内容。

```python
# 错误：无文件头
def create_vm(params):

  - 删除操作不可逆，调用前需由上层做权限校验

"""
from typing import List

```

```go
package vm
```

**STATIC-NAME-05** `[强制]` 文件名必须与文件中的主导出类名（或主要职责）保持一致；单个文件只定义一个主要类/模块，辅助类/函数不得超过主类规模。

违规信号：`user_service.py` 中主类命名为 `AccountManager`；`vm.go` 中主结构体为 `VirtualMachineHandler`；一个文件定义了 3 个同等重要的类。

```python
# 错误：文件名与主类名不一致
# 文件名：user_service.py
class AccountManager:           # 类名应为 UserService
    def create_user(self): ...

# 错误：文件名含糊，主类名也含糊
# 文件名：manager.py
class DataManager:              # manager 是什么 manager？
    ...

# 正确：文件名 = 主类名（蛇形 vs 驼峰转换）
# 文件名：user_service.py
class UserService:              # snake_case ↔ PascalCase 对应
    def create_user(self): ...

# 文件名：vm_driver.py
class VMDriver:
    def create_vm(self): ...
```

```go
// 错误：文件名与主结构体不一致
// 文件名：handler.go
type VirtualMachineController struct { ... }   // 应与文件名对应

// 正确
// 文件名：vm_handler.go
type VMHandler struct { ... }

// 文件名：vm_service.go
type VMService struct { ... }
```

对应关系规则：
- Python：`user_service.py` → `UserService`（snake_case → PascalCase）
- Go：`vm_handler.go` → `VMHandler`（snake_case → PascalCase，缩写保持大写）

对应关系规则：
- Python：`user_service.py` → `UserService`（snake_case → PascalCase）
- Go：`vm_handler.go` → `VMHandler`（snake_case → PascalCase，缩写保持大写）

**STATIC-NAME-06** `[强制]` 禁止使用非标准缩写；允许使用的缩写必须在项目中维护一份白名单列表，AI 命名时严格参考此列表，不在白名单内的词必须使用完整拼写。

违规信号：`usr` 代替 `user`；`mgr` 代替 `manager`；`cfg` 代替 `config`；`ctx` 在未列入白名单的项目中使用；单字符变量（非循环索引）作为业务变量。

项目级允许缩写白名单（各项目可在此基础上增减，增减需更新此列表）：

| 缩写 | 完整词 | 说明 |
|------|--------|------|
| `id` | identifier | 资源唯一标识 |
| `url` | URL | 统一资源定位符 |
| `api` | API | 应用程序接口 |
| `ctx` | context | 请求上下文 |
| `db` | database | 数据库 |
| `req` | request | 请求对象 |
| `resp` | response | 响应对象 |
| `err` | error | 错误对象（Go 惯例） |
| `ip` | IP address | IP 地址 |
| `az` | availability zone | 可用区 |
| `vm` | virtual machine | 虚拟机 |
| `cpu` | CPU | 中央处理器 |
| `uid` | user id | 用户 ID |
| `pid` | project id | 项目 ID |
| `msg` | message | 消息（仅日志场景） |
| `buf` | buffer | 缓冲区（仅底层IO场景） |

```python
# 错误：不在白名单内的缩写
❌ usr = get_user()              # usr → user
❌ mgr = UserManager()          # mgr → manager
❌ cfg = load_config()          # cfg → config（config 已是简洁词，无需缩写）
❌ perm = get_permission()      # perm → permission
❌ evt = Event()                # evt → event
❌ svr = Server()               # svr → server

# 正确：完整拼写或白名单内缩写
✅ user = get_user()
✅ manager = UserManager()
✅ config = load_config()
✅ permission = get_permission()
✅ ctx = request.context        # ctx 在白名单内
✅ vm_id = params.get("id")     # vm、id 均在白名单内
```

```go
// 错误
❌ usr := getUser()
❌ mgr := newManager()
❌ svr := &Server{}

// 正确
✅ user := getUser()
✅ manager := newManager()
✅ server := &Server{}
✅ ctx := context.Background()  // ctx 在白名单内
✅ err := doSomething()         // err 在白名单内（Go 惯例）
```

**STATIC-NAME-07** `[强制]` 状态值、类型值等有限取值集合必须使用枚举（`Enum`）定义，禁止在业务逻辑中直接使用字符串字面量表示状态/类型；枚举定义使代码的所有可能取值对 AI 和人类均可读可追踪。

> 此规则是 STATIC-NAME-03（禁止魔法字符串）在状态/类型场景的强化版：不仅要提取为常量，更要用 Enum 表达有限取值语义。

违规信号：代码中出现 `status == "running"`、`type == "mysql"`、`role == "admin"` 等字符串比较；常量散落在多处而非集中在枚举类；新增状态时需全局搜索替换。

```python
# 错误：字符串字面量表示状态
❌ if vm["status"] == "running":
       ...
❌ if user.role == "admin":
       ...
❌ if db_type == "mysql" or db_type == "postgres":
       ...

# 正确：Enum 定义所有可能取值
from enum import Enum

class VMStatus(Enum):
    RUNNING   = "running"
    STOPPED   = "stopped"
    CREATING  = "creating"
    DELETING  = "deleting"
    ERROR     = "error"

class UserRole(Enum):
    ADMIN     = "admin"
    MSP       = "msp"
    TENANT    = "tenant"
    MEMBER    = "member"

class DBType(Enum):
    MYSQL     = "mysql"
    POSTGRES  = "postgres"
    REDIS     = "redis"

# 使用枚举比较
✅ if vm["status"] == VMStatus.RUNNING.value:
       ...

✅ if user.role == UserRole.ADMIN.value:
       ...

✅ if db_type in (DBType.MYSQL.value, DBType.POSTGRES.value):
       ...

# 枚举的额外好处：AI 可通过枚举定义理解所有可能状态
# VMStatus 的所有值：RUNNING, STOPPED, CREATING, DELETING, ERROR
# 无需全局搜索字符串即可了解完整状态机
```

```go
// 错误：字符串字面量
❌ if vm.Status == "running" { ... }
❌ if user.Role == "admin" { ... }

// 正确：类型常量 + iota 或字符串常量组
type VMStatus string
const (
    VMStatusRunning  VMStatus = "running"
    VMStatusStopped  VMStatus = "stopped"
    VMStatusCreating VMStatus = "creating"
    VMStatusDeleting VMStatus = "deleting"
    VMStatusError    VMStatus = "error"
)

type UserRole string
const (
    UserRoleAdmin  UserRole = "admin"
    UserRoleMSP    UserRole = "msp"
    UserRoleTenant UserRole = "tenant"
    UserRoleMember UserRole = "member"
)

✅ if vm.Status == VMStatusRunning { ... }
✅ if user.Role == UserRoleAdmin { ... }
```

枚举定义位置规范：
- Python：在 `constants.py` 或对应模块的 `enums.py` 中定义
- Go：在对应包的 `constants.go` 中定义，或与类型定义同文件

### 复杂度

**STATIC-CC-01** `[建议]` 单个函数的圈复杂度（Cyclomatic Complexity）不超过 10；超过 15 为高风险，必须重构。

违规信号：函数内 `if/elif/for/while/try/except/and/or` 分支过多；`radon cc` / `gocyclo` 输出等级 C（CC=8-10）及以上。

```python
# 圈复杂度约 12，过高
def process(data, flag, mode, retry):
    if flag:
        if mode == "A":
            for item in data:
                if item > 0:
                    try:
                        pass
                    except ValueError:
                        pass
                    except TypeError:
                        pass
                elif item == 0:
                    pass
        elif mode == "B":
            pass
    else:
        pass

# 拆分为职责单一的小函数，每个 CC 不超过 5
def process(data, flag, mode, retry):
    if not flag:
        return _handle_no_flag(data)
    return _handle_by_mode(data, mode, retry)
```

**STATIC-CC-02** `[建议]` 单个函数的认知复杂度不超过 15；函数最大嵌套深度不超过 3 层；单个源文件累积圈复杂度不超过 50。

违规信号：`if` 内嵌 `for` 内嵌 `if` 内嵌 `try` 超过 3 层；`pylint` 报 `too-complex`。

```python
# 4 层嵌套，超出限制
def parse(data):
    for group in data:
        for item in group:
            if item.valid:
                if item.active:   # 第 4 层
                    process(item)

# 提取子函数降低嵌套
def parse(data):
    for group in data:
        _process_group(group)

def _process_group(group):
    for item in group:
        if item.valid and item.active:
            process(item)
```

### 重复代码

**STATIC-DUP-01** `[建议]` 连续重复代码块 ≥ 5 行时必须提取为函数；代码库整体重复率不超过 5%；相同代码片段出现 ≥ 3 次时建议重构。

违规信号：复制粘贴的代码块仅变量名不同；`pylint` 报 `duplicate-code`；SonarQube 重复率 > 5%。

```python
# 相同逻辑出现 3 处，仅资源类型不同
def validate_vm(vm):
    if not vm.get("id"):
        raise ValueError("id is required")
    if not vm.get("name"):
        raise ValueError("name is required")
    if len(vm.get("name", "")) > 64:
        raise ValueError("name too long")

def validate_disk(disk):
    if not disk.get("id"):
        raise ValueError("id is required")
    if not disk.get("name"):
        raise ValueError("name is required")
    if len(disk.get("name", "")) > 64:
        raise ValueError("name too long")

# 提取通用校验函数
def _validate_resource(resource, resource_type):
    if not resource.get("id"):
        raise ValueError("{} id is required".format(resource_type))
    name = resource.get("name", "")
    if not name:
        raise ValueError("{} name is required".format(resource_type))
    if len(name) > 64:
        raise ValueError("{} name too long".format(resource_type))

def validate_vm(vm):
    _validate_resource(vm, "vm")

def validate_disk(disk):
    _validate_resource(disk, "disk")
```

**STATIC-DUP-02** `[强制]` 禁止重复的模板代码（Boilerplate）；相似结构必须通过抽象（基类/工厂/装饰器/模板方法）消除，而非复制粘贴后微调。

违规信号：多个 handler/provider/controller 函数有相同的"校验→查询→处理→返回"骨架但逐一手写；多个 CRUD 接口重复相同的参数解析、权限检查、日志记录代码。

```python
# 多个接口重复相同的模板骨架（参数解析 + 权限 + 日志）
def create_vm(self, req, body):
    params = schema.validate(body)           # 模板代码
    check_permission(req, "create_vm")       # 模板代码
    LOG.info("create vm, params=%s", params) # 模板代码
    result = self._driver.create(params)
    return result

def delete_vm(self, req, body):
    params = schema.validate(body)           # 重复
    check_permission(req, "delete_vm")       # 重复
    LOG.info("delete vm, params=%s", params) # 重复
    result = self._driver.delete(params)
    return result

# 用装饰器或基类模板消除重复骨架
def _action(action_name):
    """统一处理校验、权限检查、日志的装饰器"""
    def decorator(fn):
        def wrapper(self, req, body):
            params = schema.validate(body)
            check_permission(req, action_name)
            LOG.info("%s, params=%s", action_name, params)
            return fn(self, req, params)
        return wrapper
    return decorator

@_action("create_vm")
def create_vm(self, req, params):
    return self._driver.create(params)

@_action("delete_vm")
def delete_vm(self, req, params):
    return self._driver.delete(params)
```

```go
// 多个接口重复相同的模板骨架
func (h *Handler) CreateVM(ctx context.Context, req *CreateVMReq) (*VM, error) {
    if err := validate(req); err != nil { return nil, err }   // 重复
    if err := checkPerm(ctx, "create_vm"); err != nil { return nil, err } // 重复
    log.Infof("create vm, req=%v", req)                       // 重复
    return h.driver.Create(ctx, req)
}

func (h *Handler) DeleteVM(ctx context.Context, req *DeleteVMReq) error {
    if err := validate(req); err != nil { return err }        // 重复
    if err := checkPerm(ctx, "delete_vm"); err != nil { return err } // 重复
    log.Infof("delete vm, req=%v", req)                       // 重复
    return h.driver.Delete(ctx, req.ID)
}

// 用中间件/高阶函数封装公共骨架
func withValidateAndAuth(action string, fn func(ctx context.Context) error) func(ctx context.Context) error {
    return func(ctx context.Context) error {
        if err := checkPerm(ctx, action); err != nil {
            return err
        }
        log.Infof("action=%s", action)
        return fn(ctx)
    }
}
```

### 长度限制

**STATIC-LEN-01** `[建议]` 单个函数不超过 50 行；单个源文件不超过 500 行；单个类/模块不超过 300 行；函数参数不超过 4 个，超过时用对象/结构体封装。

### 继承与组合

**STATIC-INHERIT-01** `[建议]` 优先使用组合（Composition）替代继承（Inheritance）；类继承层级不超过 2 层（不含语言内置基类如 `object`/`Exception`）。

违规信号：继承链超过 2 层（`A → B → C → D`）；子类仅为复用父类方法而非表达"is-a"关系；父类被多个不相关子类继承导致职责膨胀。

```python
# 继承链 3 层，且 SpecialAdmin 与 User 并非纯粹的"is-a"关系
class Base:
    def save(self): ...

class User(Base):         # 第 1 层
    def login(self): ...

class Admin(User):        # 第 2 层
    def manage(self): ...

class SpecialAdmin(Admin):  # 第 3 层，超出限制
    def audit(self): ...

# 改为组合：SpecialAdmin 持有 Admin 实例，复用其能力
class SpecialAdmin:
    def __init__(self, admin: Admin):
        self._admin = admin        # 组合，而非继承

    def manage(self):
        return self._admin.manage()

    def audit(self): ...
```

```go
// 继承链 3 层（通过嵌套结构体模拟）
type Base struct{}
type User struct{ Base }           // 第 1 层
type Admin struct{ User }          // 第 2 层
type SpecialAdmin struct{ Admin }  // 第 3 层，超出限制

// 改为组合
type SpecialAdmin struct {
    admin *Admin   // 组合，持有引用
}
func (s *SpecialAdmin) Manage() { s.admin.Manage() }
func (s *SpecialAdmin) Audit()  { ... }
```

何时可以使用继承：
- 子类与父类存在明确的"is-a"语义关系（`Dog is-a Animal`）
- 继承用于实现接口/抽象类（如 `Exception`、`BaseView`），层级不超过 2 层
- 框架要求强制继承（如 Phoenix 的 `Controller`）

### 架构约束

**STATIC-ARCH-01** `[强制]` 禁止模块间循环依赖；模块依赖关系必须是有向无环图（DAG）；底层模块不得反向依赖上层业务模块。

违规信号：`module A` 导入 `module B`，`module B` 又导入 `module A`；工具/基础库模块导入了业务层模块；`isort`/`pylint` 报循环导入错误。

```python
# 循环依赖
# user.py
from order import Order          # user 依赖 order

# order.py
from user import User            # order 反过来依赖 user → 循环

# 解决方案 1：提取公共数据模型到第三个模块
# models.py（无依赖）
class User: ...
class Order: ...

# user.py
from models import User, Order   # 依赖 models，不依赖 order

# order.py
from models import User, Order   # 依赖 models，不依赖 user

# 解决方案 2：依赖注入，延迟导入
# order.py
class Order:
    def get_user(self, user_id: str):
        from user import User    # 延迟导入，打破循环（临时方案，优先用方案 1）
        return User.get(user_id)
```

```go
// 循环依赖（Go 编译器会直接报错）
// package user
import "myapp/order"   // user 依赖 order

// package order
import "myapp/user"    // order 依赖 user → 编译失败

// 解决：提取公共接口到独立包
// package types（无其他内部依赖）
type UserID string
type OrderID string

// package user
import "myapp/types"   // 只依赖 types

// package order
import "myapp/types"   // 只依赖 types
```

违规信号：一个文件包含多个互不相关的类/函数；`utils.py` 中混有字符串处理、时间转换、网络请求等完全不同领域的代码；类承担多个不相关职责导致频繁被不同模块修改。

- 框架约定的文件名（如 Django 的 `views.py`、`models.py`）

**STATIC-ARCH-03** `[强制]` API 层禁止直接返回裸字典（`dict`/`map`），必须使用 DTO（Data Transfer Object）、`Request`/`Response` 等专用数据类型定义接口数据结构；接口的输入输出字段必须可被静态分析工具和 AI 追踪。

违规信号：Controller/Handler 直接 `return {"code": 0, "data": {...}}`；函数返回类型为 `dict` 或 `interface{}`；接口字段无类型约束，调用方只能靠文档或猜测。

```python

❌ def create_vm(self, req, body):
       vm = self.driver.create(body)
       return {"id": vm.id, "name": vm.name, "status": vm.status}  # 裸字典

❌ def list_vms(self, req) -> dict:   # 返回类型为 dict，字段不可知
       vms = self.driver.list()
       return {"data": [v.__dict__ for v in vms], "total": len(vms)}

# 正确：DTO/Response 类定义，字段显式可见
from dataclasses import dataclass
from typing import List

@dataclass
class VMResponse:

    name: str
    status: VMStatus
    az_id: str
    created_at: str

class ListVMResponse:
    data: List[VMResponse]

def create_vm(self, req, body: CreateVMRequest) -> VMResponse:
    vm = self.driver.create(body)
    return VMResponse(
        id=vm.id,
        name=vm.name,
        status=VMStatus(vm.status),
        az_id=vm.az_id,

def list_vms(self, req) -> ListVMResponse:

    return ListVMResponse(
        data=[VMResponse(...) for v in vms],
```

**STATIC-ARCH-04** `[强制]` 所有公共 API 函数（对外暴露的 Controller/Service/Provider 方法）必须有完整的显式类型签名（参数类型 + 返回类型），禁止使用 `Any`、裸 `dict`、`interface{}` 等宽泛类型作为签名；AI 和静态分析工具必须可通过签名推断接口契约。

```python
       ...

❌ def list_vms(self, req):          # 返回类型缺失

from typing import List
from myapp.request import CreateVMRequest

✅ def _fetch_vm_detail(self, vm_id: str) -> Optional[VMDetail]:
       ...
```

### 元编程限制

**STATIC-META-01** `[建议]` 避免使用 `__getattr__`、动态 `import`、`exec()`、`eval()` 等元编程手段；如确实需要，必须同时提供人类和 AI 均可读的**静态注册表**（显式映射字典或常量列表），确保依赖关系可被静态分析工具追踪。

违规信号：使用 `__getattr__` 动态路由方法调用；`importlib.import_module` 拼接模块名；`exec()`/`eval()` 执行动态生成的代码字符串；无任何静态注册表可供查阅。

```python
# 动态 import，AI 和静态分析工具无法追踪依赖
def get_driver(driver_type):
    module = importlib.import_module(
        "myapp.drivers.{}".format(driver_type)   # 依赖关系不可见
    )
    return module.Driver()

# __getattr__ 动态路由，调用链不可见
class PluginManager:
    def __getattr__(self, name):
        return self._plugins[name]   # 哪些 plugin 存在？无法静态判断

# exec 执行动态代码，安全风险且不可读
exec("result = {} + {}".format(a, b))
```

```python
# 必须同时提供静态注册表（显式映射字典）
# 即使使用了动态 import，也要在同一文件维护一份可读的注册表

# 静态注册表（人类和 AI 均可读）
DRIVER_REGISTRY = {
    "mysql":   "myapp.drivers.mysql.MySQLDriver",
    "postgres": "myapp.drivers.postgres.PostgresDriver",
    "redis":   "myapp.drivers.redis.RedisDriver",
}

def get_driver(driver_type: str):
    if driver_type not in DRIVER_REGISTRY:
        raise ValueError("unknown driver: {}".format(driver_type))
    # 动态 import 仍可接受，但 DRIVER_REGISTRY 提供了可读的全量映射
    module_path, class_name = DRIVER_REGISTRY[driver_type].rsplit(".", 1)
    module = importlib.import_module(module_path)
    return getattr(module, class_name)()
```

```python
# 插件系统：__getattr__ 改为显式注册 + 静态字典
class PluginManager:
    # 静态注册表，所有插件显式列出
    _PLUGIN_REGISTRY: dict = {
        "alarm":   AlarmPlugin,
        "report":  ReportPlugin,
        "backup":  BackupPlugin,
    }

    def get(self, name: str):
        if name not in self._PLUGIN_REGISTRY:
            raise KeyError("plugin {} not registered".format(name))
        return self._PLUGIN_REGISTRY[name]()
```

何时可以使用元编程：
- 框架层代码（如路由装饰器、ORM 字段描述符），需在 README 中说明
- 必须配套完整的静态注册表，注册表与动态代码在同一文件或相邻文件
- 有充分注释解释动态行为的范围和边界

### 安全静态检查

**STATIC-SEC-01** `[建议]` 静态分析必须检测以下安全问题；Python 使用 `bandit`，Go 使用 `gosec`，发现问题必须修复。

| 安全问题 | 违规示例 | 修复方向 |
|---------|---------|---------|
| SQL 注入 | 字符串拼接 SQL | 参数化查询 |
| 命令注入 | `os.system(user_input)` | 参数列表调用 |
| 不安全反序列化 | `yaml.load()` / `pickle.loads()` | `yaml.safe_load()` |
| 硬编码敏感信息 | `password = "admin123"` | 环境变量/配置中心 |
| 缺失输入验证 | 外部输入直接使用 | 校验后再使用 |
| 不安全随机数 | `random.random()` 用于密码学 | `os.urandom()` |

```python
# bandit 会标记的典型问题
# 错误
data = yaml.load(content)             # B506: 不安全的 yaml.load
os.system("rm -rf " + user_path)      # B605: 命令注入
SECRET = "abc123def456"               # B105: 硬编码密码

# 正确
data = yaml.safe_load(content)
subprocess.run(["rm", "-rf", user_path])
SECRET = os.environ.get("SECRET_KEY")
```

### 性能静态检查

**STATIC-PERF-01** `[建议]` 静态分析应识别并标注以下潜在性能问题：

| 问题类型 | 违规信号 | 建议 |
|---------|---------|------|
| 循环内 DB 查询（N+1） | `for` 循环内调用 DB/API | 改为批量查询 |
| 循环内字符串拼接 | `result += string` 在循环内 | 使用 `join` |
| 不必要的全量加载 | 无分页的大表查询 | 强制分页 |
| 高复杂度算法 | O(n²) 嵌套循环操作大数据集 | 优化算法或数据结构 |
| 频繁序列化 | 循环内 `json.loads()`/`json.dumps()` | 提取到循环外 |

```python
# N+1 查询，每次循环一次 DB
for vm_id in vm_ids:
    vm = db.query("SELECT * FROM vms WHERE id = %s", vm_id)

# 批量查询，一次 DB
vms = db.query("SELECT * FROM vms WHERE id IN %s", tuple(vm_ids))

# 循环内字符串拼接 O(n²)
result = ""
for item in items:
    result += str(item)

# join O(n)
result = "".join(str(item) for item in items)
```

## Code Review Checklist

逐项扫描，违规输出 `[规则ID] <文件>:<行号> — <描述>`。

**命名语义**
- [ ] STATIC-NAME-01 — 布尔变量/函数是否使用 `is_`/`has_`/`can_` 等语义前缀
- [ ] STATIC-NAME-02 — 函数名是否为动词+宾语形式；是否存在 `handle`/`process`/`do` 等泛化命名
- [ ] STATIC-NAME-03 — 常量是否全大写下划线命名；代码中是否有魔法数字/字符串未提取为常量
- [ ] STATIC-NAME-04 — 新建文件头部是否有模块级 docstring 说明职责和注意事项
- [ ] STATIC-NAME-05 — 文件名是否与主导出类名一致（snake_case ↔ PascalCase）
- [ ] STATIC-NAME-06 — 是否使用了不在白名单内的缩写（如 `usr`/`mgr`/`cfg`/`svr`）；完整词是否拼写正确
- [ ] STATIC-NAME-07 — 状态值/类型值是否使用 Enum 定义；业务逻辑中是否直接出现状态字符串字面量

**复杂度**
- [ ] STATIC-CC-01 — 函数圈复杂度是否不超过 10；CC > 15 必须重构
- [ ] STATIC-CC-02 — 嵌套深度是否不超过 3 层；单文件累积 CC 是否不超过 50（建议）

**重复代码**
- [ ] STATIC-DUP-01 — 是否有 5 行以上重复代码未提取；相同片段是否出现 3 次以上
- [ ] STATIC-DUP-02 — 是否有重复的模板骨架（校验→权限→日志→处理）未通过装饰器/基类抽象

**长度限制**
- [ ] STATIC-LEN-01 — 函数是否不超过 50 行；文件是否不超过 500 行；参数是否不超过 4 个

**可维护性**
- [ ] STATIC-MAINT-01 — 可维护性指数是否不低于 65；技术债务比率是否不超过 5%（建议）
- [ ] STATIC-MAINT-02 — TODO/FIXME 是否有负责人和截止时间
- [ ] STATIC-MAINT-03 — 配置文件是否集中在 `/config` 目录；是否有配置散落在业务/测试/根目录

**测试覆盖率**
- [ ] STATIC-COV-01 — 项目覆盖率是否不低于 80%；3-High 函数是否不低于 90%（建议）
- [ ] STATIC-COV-02 — 异常处理分支、回滚逻辑、边界条件是否有测试（建议）

**继承与组合**
- [ ] STATIC-INHERIT-01 — 类继承层级是否不超过 2 层；是否存在组合比继承更合适的场景（建议）

**架构约束**
- [ ] STATIC-ARCH-01 — 是否存在模块间循环依赖；底层模块是否反向依赖了上层业务模块
- [ ] STATIC-ARCH-02 — 文件/类/函数是否遵循单一职责；是否存在 `utils.py`/`helpers.go` 等泛化文件名作为杂物箱
- [ ] STATIC-ARCH-03 — API 层是否直接返回裸字典/map；接口输入输出是否使用专用 DTO/Request/Response 类型
- [ ] STATIC-ARCH-04 — 公共 API 函数是否有完整显式类型签名（参数+返回值）；是否使用 `Any`/`dict`/`interface{}` 等宽泛类型

**元编程限制**
- [ ] STATIC-META-01 — 是否使用了 `__getattr__`/动态 `import`/`exec()`/`eval()`；如使用是否有静态注册表（建议）

**安全**
- [ ] STATIC-SEC-01 — 是否有 SQL 注入/命令注入/不安全反序列化/硬编码密码/缺失输入验证

**性能**
- [ ] STATIC-PERF-01 — 是否有循环内 DB 查询（N+1）；循环内字符串拼接；无分页大表查询
