## 模块独立性
- **[强制]** 独立开发、测试、运行、部署、升级和维护模块  
- **[强制]** 使模块在开发早期能独立运行以便验证  
- **[强制]** 在满足依赖并安装后，软件包能单独执行其测试用例  
- **[建议]** 将易变逻辑与不变业务逻辑分离以便单元测试

## 依赖管控
- **[强制]** 禁止出现双向依赖或环形依赖  
- **[建议]** 依赖接口而非具体实现以解耦模块  
- **[强制]** 尽量减少对其它模块的依赖以保证基本运行并便于快速替换

## 错误隔离
- **[强制]** 在模块范围内捕捉并处理错误，除 API 明确外不向外泄漏  
- **[强制]** 在模块故障终止后自动恢复并恢复到终止前或确定的工作状态

## 接口设计
- **[强制]** 只暴露必须的最少接口  
- **[强制]** 将接口开放范围控制到尽可能小，优先本机其次局域网  
- **[强制]** 默认只使用最小权限，仅在必要时使用更高权限

## 单元测试覆盖率
- **[强制]** 单元测试行覆盖率达到 `80%`

## 版本兼容性
- **[强制]** 以包为单位进行版本管理，大版本号作为兼容版本号  
- **[强制]** 保证相同大版本号的正式版本向前兼容并能替换旧版本 `mypack-2.4`  
- **[强制]** 保证大版本相同的软件包的测试案例在较新版本上可执行
