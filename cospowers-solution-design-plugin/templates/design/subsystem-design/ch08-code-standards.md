# 8. **代码规范和质量要求**

## 8.1. **代码规范**

| 规范类型 | 要求 | 工具检查 |
|---|---|---|
| Python代码规范 | PEP 8 | flake8, black |
| 命名规范 | 见《编码规范》 | pylint |
| 注释规范 | Docstring必须 | pydocstyle |
| 类型注解 | 关键函数必须 | mypy |

## 8.2. **日志要求**

<span style="color:orange">**[填写要求] 日志不仅是运维工具，也是代码质量的一部分。必须在编码阶段明确日志规范，避免事后补日志。**</span>

| 要求类型 | 具体要求 | 说明 |
|---|---|---|
| 日志框架 | *统一使用项目指定的日志库，禁止 print 输出* | *如 Python logging / Go zap* |
| 请求链路追踪 | *所有日志必须携带 request_id / trace_id* | *便于跨服务追踪问题* |
| 入参出参记录 | *API入口和外部调用须记录关键入参和返回状态* | *DEBUG级别记录完整参数，INFO级别记录摘要* |
| 异常日志 | *catch异常时必须记录ERROR日志，包含异常堆栈* | *禁止吞掉异常不记录* |
| 敏感信息脱敏 | *密码、Token、身份证号等敏感字段必须脱敏后记录* | *使用统一脱敏工具函数* |
| 日志级别使用 | *严格按级别使用：DEBUG调试/INFO正常操作/WARNING异常但可恢复/ERROR失败需关注/CRITICAL系统级故障* | *禁止ERROR级别滥用* |

**日志格式要求:**

```
[时间戳] [级别] [模块名] [request_id] [user_id] 消息内容 {结构化字段}
```

## 8.3. **操作审计要求**

<span style="color:orange">**[填写要求] 所有涉及数据变更的操作必须记录审计日志，审计日志与业务日志分离存储。**</span>

| 要求类型 | 具体要求 | 说明 |
|---|---|---|
| 审计范围 | *创建、修改、删除等写操作必须记录* | *读操作按需记录（如敏感数据查询）* |
| 审计字段 | *操作人、操作时间、操作类型、操作对象、操作前后值、请求来源IP* | *字段不可缺省* |
| 存储方式 | *写入审计日志表 / 推送到审计服务* | *说明具体实现方式* |
| 不可篡改 | *审计日志只允许追加写入，禁止修改和删除* | *审计表不提供UPDATE/DELETE接口* |
| 保留时长 | *至少保留180天* | *按合规要求调整* |

**审计日志结构示例:**

```json
{
  "audit_id": "aud-20260324-001",
  "timestamp": "2026-03-24T10:00:00Z",
  "operator": {"user_id": "user-001", "user_name": "zhangsan", "source_ip": "10.0.1.100"},
  "action": "CREATE",
  "resource": {"type": "service", "id": "svc-001", "name": "my-service"},
  "before": null,
  "after": {"name": "my-service", "status": "creating", "config": "..."},
  "result": "SUCCESS",
  "request_id": "req-12345"
}
```

## 8.4. **权限配置要求**

<span style="color:orange">**[填写要求] 明确本子系统涉及的权限点，包括API权限、数据权限、功能权限等。权限点必须与API接口清单（§3.1）对应。**</span>

### 8.4.1. **权限点清单**

| 权限编号 | 权限名称 | 权限类型 | 关联接口/功能 | 默认角色 | 说明 |
|---|---|---|---|---|---|
| PERM-001 | *service:create* | API权限 | *POST /api/v1/services* | *管理员* | *创建服务* |
| PERM-002 | *service:read* | API权限 | *GET /api/v1/services* | *普通用户、管理员* | *查看服务列表和详情* |
| PERM-003 | *service:update* | API权限 | *PUT /api/v1/services/{id}* | *管理员* | *修改服务配置* |
| PERM-004 | *service:delete* | API权限 | *DELETE /api/v1/services/{id}* | *管理员* | *删除服务* |

### 8.4.2. **角色与权限映射**

| 角色 | 包含的权限 | 说明 |
|---|---|---|
| *超级管理员* | *所有权限* | *系统最高权限* |
| *管理员* | *PERM-001 ~ PERM-004* | *服务全生命周期管理* |
| *普通用户* | *PERM-002* | *仅查看* |

### 8.4.3. **数据权限**

<!--
【填写要求】
如果涉及数据隔离（如多租户、多项目），需说明数据权限的控制粒度和实现方式。
不涉及则写明"不涉及"。
-->

| 数据权限类型 | 控制粒度 | 实现方式 | 说明 |
|---|---|---|---|
| *租户隔离* | *租户级* | *数据库行级过滤（tenant_id字段）* | *用户只能访问本租户的数据* |
| *项目隔离* | *项目级* | *查询条件自动注入 project_id* | *用户只能访问所属项目的数据* |

### 8.4.4. **权限校验方式**

| 要求项 | 说明 |
|---|---|
| 校验位置 | *API网关统一校验 / 服务内校验 / 两者结合* |
| 校验时机 | *每次请求必须校验，不可跳过* |
| 权限缓存 | *是否缓存权限数据、缓存策略和失效时间* |
| 越权防护 | *水平越权：校验资源归属；垂直越权：校验角色权限* |
| 未授权响应 | *未认证返回401，无权限返回403，含具体缺少的权限点* |

## 8.5. **代码质量要求**

| 质量指标 | 要求 | 检查方式 |
|---|---|---|
| 单元测试覆盖率 | > 80% | pytest-cov |
| 代码复杂度 | McCabe < 10 | radon |
| 代码重复率 | < 5% | pylint |
| 静态代码分析 | 无严重问题 | SonarQube |
