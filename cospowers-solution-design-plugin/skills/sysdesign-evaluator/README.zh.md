# Sysdesign Evaluator — 中文参考说明

> 本文件仅供人阅读，不加入 AI 上下文。AI 使用的是同目录下的 `SKILL.md`（英文）。

## 概述

对 `design-spec` 产出的系统设计文档（`*-system-design.md`）和 OpenAPI 规范（`*-openapi.yaml`）进行逐条规范扫描和质量打分。支持单文档或双文档评估，自动识别设计子类型并匹配 M1-M6 审查模块。跨文档一致性由 `doc-quality-evaluator` 负责。

## 审查流程

```
Phase 0: 加载（自动发现文档 → 识别设计子类型 → 确定审查模块 → 加载规范）
  → Phase 1: 逐条规范扫描（按适用模块逐项检查 + 12 条 SYS- 内置项 + 3 条红线）
  → Phase 2: 跳过（跨文档一致性由 doc-quality-evaluator 承担）
  → Phase 3: 加权打分
  → Phase 4: 生成评估报告
```

复杂文档可选 **3-Stage 并行审查流程**：Stage 1 识别类型 → Stage 2 确定模块 → Stage 3 为每个模块启动子代理并行审查。

## 设计子类型自动识别

三级策略自动判断文档子类型（总体设计 / 概要设计 / 技术需求设计）：
1. 文件名匹配（关键词：总体设计/overall design、概要设计/HLD、技术需求设计）
2. 内容关键词（前 200 行搜索）
3. 结构特征（章节名推断）

三级都无法识别时提示用户确认。

## M1-M6 审查模块映射

| 设计子类型 | 审查模块 |
|------|------|
| 总体设计 | M1 + M2 + M3 + M5 + M6 |
| 概要设计 | M1 + M4 + M5 + M6 |
| 技术需求设计 | M1 + M2 + M5 + M6 |

| 模块 | 全称 | 检查内容 |
|------|------|------|
| M1 | 文档书写规范 | 格式、术语、图表、编号、引用 |
| M2 | 技术需求设计 | 需求追溯、设计与需求映射、技术选型依据 |
| M3 | 总体设计 | 架构方案、模块划分、技术栈、系统流程 |
| M4 | 概要设计 | 函数定义、接口设计、数据结构、模块内部流程 |
| M5 | 国际化i18n设计 | 多语言支持、时区处理、字符编码 |
| M6 | 不贰过设计准则 | 历史故障经验、已知陷阱规避 |

## 内置检查项（12 条 SYS- 系列）

来源于技术需求设计评审标准 V2.0，仅适用于系统设计文档：架构差距定量化、性能测试计划完整性、系统容量边界、分布式一致性策略、灾难恢复计划、版本升级兼容性、关键路径性能目标、限流熔断策略、跨域集成安全设计、可调试性设计、可监控性设计、需求覆盖完整性。

## 评分体系

| 维度 | 权重 | 说明 |
|------|:--:|------|
| 技术可行性 | 30% | 组件真实可用，逻辑无漏洞，方案有依据 |
| 需求完整性 | 25% | 所有关键场景被覆盖，DFX 有实质内容 |
| 可测试性 | 20% | 功能流程覆盖正常/异常/边界三类场景 |
| 规范符合性 | 15% | 文档格式、命名、章节完整性 |
| 架构合理性 | 10% | 设计模式与现有架构一致，边界清晰 |

扣分：Critical -40 / Error -20 / Warning -8。

| 总分 | 等级 | 含义 |
|------|:--:|------|
| 95-100 | A | 优秀，可进入实施规划 |
| 80-94 | B | 良好，修复 Error 级问题后进入实施 |
| 65-79 | C | 勉强，修复 Error+Critical 后重新评估 |
| <65 | D | 不及格，大幅返工 |
| 红线违反 | F | 直接失败 |

## 3 条红线（一票否决）

| 红线 | 标准 |
|------|------|
| R1 章节完整性 | 缺失章节数 ≥ 3（主入口 + 7 子文档） |
| R2 可靠性章节 | 必须覆盖故障容错/资源使用/业务流程可靠性 |
| R3 安全性章节 | 必须覆盖安全基线/威胁建模/数据保护 |

> OpenAPI 规范不做红线检查。

## 派遣方式

由 `design-spec` 步骤 8（系统设计完成后）和步骤 9（OpenAPI 完成后）以隔离子代理方式派遣。评估完成后将评级、红线状态、问题表格、报告路径返回给调用方。

## 下一步

- 评级 A/B（≥ 80）→ `design-spec` 继续下一步
- 评级 C/D/F → 修复问题后重新派遣 `sysdesign-evaluator`，直至达到 B 级
