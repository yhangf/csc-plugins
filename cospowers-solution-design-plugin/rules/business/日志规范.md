# Python 日志规范

## 用途

本文档供 AI 执行两类任务：**代码生成**（写日志前对照规则）、**代码 Review**（逐条扫描输出违规）。

违规报告格式：`[规则ID] <文件>:<行号> — <描述>`

强制等级：`[强制]` 必须遵守 / `[建议]` 推荐遵守

---

## 级别决策树

```
系统无法继续运行？
  是 → CRITICAL（raise/exit 前打印）
  否 → 意外故障导致当前请求失败？
        是 → ERROR（raise 前打印，附堆栈）
        否 → 预期内已处理的异常或业务规则违规？
              是 → WARNING
              否 → 增删改入口 / 关键业务事件 / 状态变更？
                    是 → INFO
                    否 → DEBUG
```

---

## 规则详情

### 日志级别

**LOG-LVL-01** `[强制]` 系统无法继续运行的致命故障使用 CRITICAL，在 raise/exit 前打印。
违规信号：`raise SystemExit` / `sys.exit()` / 系统初始化 `except` 块前无 CRITICAL。
```python
❌ if not os.path.exists(path): raise SystemExit(1)  # 缺少 CRITICAL
✅ LOG.critical("Config missing, path=%s", path); raise SystemExit(1)
```

---

**LOG-LVL-02** `[强制]` 影响当前请求正常运行的意外故障使用 ERROR，在 raise 前打印。
适用：第三方 API 异常（含错误码）、SQLException、非业务异常。
违规信号：`except` 块中有 `raise` 但前面无 ERROR 日志。
```python
❌ except SQLException as e: raise  # 缺少 ERROR 日志
✅ except SQLException as e: LOG.error("DB query failed, ..., error=%s", str(e), exc_info=True); raise
```

---

**LOG-LVL-03** `[强制]` ERROR 不得用于可预期的业务校验，应改为 WARNING。
违规信号：业务判断（`if not valid` / `if xxx is None` / `if age < 0`）后打 ERROR。
```python
❌ if age < 0: LOG.error("Invalid age, age=%s", age)
✅ if age < 0: LOG.warning("Invalid age, age=%s", age)
```

---

**LOG-LVL-04** `[强制]` 预期内已处理的异常、业务规则违规、用户输入错误使用 WARNING。
违规信号：有 fallback/return 的 `except` 或业务 `if` 分支无 WARNING 日志。
```python
❌ if config is None: return DEFAULT_CONFIG  # 降级行为无日志
✅ if config is None: LOG.warning("Config not found, using default, userId=%s", user_id); return DEFAULT_CONFIG
```

---

**LOG-LVL-05** `[强制]` 系统/集成故障不得用 WARNING，应为 ERROR。
违规信号：`except NetworkError` / `ConnectionError` / `TimeoutError` 后打 WARNING。
```python
❌ except NetworkError as e: LOG.warning("Remote sync failed, error=%s", str(e))
✅ except NetworkError as e: LOG.error("Remote sync failed, error=%s", str(e), exc_info=True); raise
```

---

**LOG-LVL-06** `[强制]` 增删改接口入口必须打 INFO 日志，且必须包含入参。
违规信号：函数名含 `create_` / `update_` / `delete_` / `add_` / `remove_` 但函数体首部无 INFO。
```python
❌ def create_order(user_id, product_id): order = Order.create(...)  # 无入口日志
✅ def create_order(user_id, product_id): LOG.info("Create order, userId=%s, productId=%s", user_id, product_id)
```

---

**LOG-LVL-07** `[强制]` 定时任务必须打开始/结束/扫描范围 INFO；消息队列消费必须打消息内容 INFO。
违规信号：定时任务函数（`scan_` / `@cron`）或消费函数（`consume` / `on_message`）无 INFO。
```python
❌ def scan_expired_orders(): do_scan()  # 无开始/结束日志
✅ LOG.info("Scan started, scanRange=%s~%s", start, end); do_scan(); LOG.info("Scan done, count=%s", n)
```

---

**LOG-LVL-08** `[强制]` 迭代次数 > 10 的循环内禁止 INFO 及以上，用循环前后摘要替代。
违规信号：`for` 循环体内含 `LOG.info` / `LOG.warning` / `LOG.error`。
```python
❌ for order in orders: LOG.info("Processing order, orderId=%s", order.id)
✅ LOG.info("Start, count=%s", len(orders)); [process(o) for o in orders]; LOG.info("Done, failed=%s", n)
```

---

**LOG-LVL-09** `[强制]` 每秒调用 > 100 次的方法禁止 INFO 及以上。
违规信号：高频工具函数（`check_` / `validate_` / `is_` / `get_`）内含 INFO。
```python
❌ def is_token_valid(token): LOG.info("Checking token, userId=%s", user_id)  # 高频方法
✅ def is_token_valid(token): LOG.debug("Checking token, tokenPrefix=%s", token[:8])
```

---

**LOG-LVL-10** `[强制]` 生产环境禁止输出 DEBUG；日志级别必须通过环境变量控制。
违规信号：`logging.basicConfig(level=logging.DEBUG)` 未做环境判断。
```python
❌ logging.basicConfig(level=logging.DEBUG)
✅ logging.basicConfig(level=logging.DEBUG if os.getenv("ENV") == "dev" else logging.INFO)
```

---

### 消息格式

**LOG-FMT-01** `[强制]` 使用 `%s` 占位符延迟格式化，禁止 f-string / 字符串拼接 / `%` 运算符预格式化 / `extra=` / kwargs。
违规信号：`LOG.xxx(f"..."` / `"..." + str(x)` / `"..." % (x,)` / `LOG.xxx("...", key=val)`。
```python
❌ LOG.info(f"User login userId={user_id}")
❌ LOG.info("User login " + str(user_id))
❌ LOG.info("User login userId=%s" % user_id)
❌ LOG.info("User login", userId=user_id)
✅ LOG.info("User login, userId=%s", user_id)
```

---

**LOG-FMT-02** `[强制]` 消息内键名使用 camelCase，禁止 snake_case 或 PascalCase。
违规信号：日志消息中出现 `user_id=` / `order_id=` / `UserId=`。
```python
❌ LOG.info("Order paid, order_id=%s, user_id=%s", order_id, user_id)
✅ LOG.info("Order paid, orderId=%s, userId=%s", order_id, user_id)
```

---

**LOG-FMT-03** `[强制]` 消息使用英语，包含足够上下文（5W1H：what/who/where/why/how）。
违规信号：消息为单词/缩写（`"failed"` / `"done"` / `"err"`）或无任何参数。
```python
❌ LOG.warning("Login failed")
✅ LOG.warning("User login failed, wrong password, userId=%s, ip=%s, attempts=%s", user_id, ip, n)
```

---

**LOG-FMT-04** `[强制]` 禁止打印 None 变量或与当前操作无关的变量。
违规信号：日志参数中含 `None` 字面量或变量名与函数业务无关。
```python
❌ LOG.info("Processing order, orderId=%s, debug=%s", order_id, None)
✅ LOG.info("Processing order, orderId=%s", order_id)
```

---

### 参数约定

**LOG-PARAM-01** `[强制]` HTTP 请求日志必须包含：`method` / `path` / `statusCode` / `duration` / `userId` / `traceId`
```python
❌ LOG.info("Request handled, method=%s, path=%s", method, path)  # 缺 duration/traceId
✅ LOG.info("HTTP handled, method=%s, path=%s, statusCode=%s, duration=%.3fs, userId=%s, traceId=%s", ...)
```

---

**LOG-PARAM-02** `[强制]` DB 操作日志必须包含：`table` / `duration` / `rowCount` / `database`
```python
❌ LOG.info("DB query done, table=users")  # 缺 rowCount/duration
✅ LOG.info("DB query done, table=users, database=main, rowCount=%s, duration=%.3fs", n, t)
```

---

**LOG-PARAM-03** `[强制]` 文件操作日志必须包含：`filename` / `path` / `operation` / `size` / `permissions`

---

**LOG-PARAM-04** `[强制]` 业务操作日志必须包含：`userId` / `resourceId` / `action` / `result` / `duration`

---

### 敏感信息

脱敏规则速查：

| 字段 | 脱敏格式 | 示例 |
|------|---------|------|
| 密码 | 全替换 | `***` |
| Token/密钥 | 前8+后4 | `sk-1234****abcd` |
| 手机号 | 中间4位 | `138****5678` |
| 信用卡 | 后4位 | `****1234` |
| 邮箱 | 建议 | `user***@domain.com` |

**LOG-SEC-01** `[强制]` 密码不得出现在日志中，必须替换为 `***`。
违规信号：变量名含 `password` / `passwd` / `pwd` 出现在日志参数中且未脱敏。

**LOG-SEC-02** `[强制]` Token/密钥脱敏为前8位+`****`+后4位。
违规信号：变量名含 `token` / `secret` / `api_key` 出现在日志参数中且未调用脱敏函数。

**LOG-SEC-03** `[强制]` 手机号脱敏为 `138****5678`（中间4位替换）。

**LOG-SEC-04** `[强制]` 信用卡号脱敏为 `****1234`（仅保留后4位）。

**LOG-SEC-05** `[建议]` 邮箱脱敏为 `user***@domain.com`。

```python
❌ LOG.info("User login, password=%s, token=%s", password, token)  # [LOG-SEC-01][LOG-SEC-02]
✅ LOG.info("User login, password=***, tokenPrefix=%s", mask_token(token))
```

### 性能

**LOG-PERF-01** `[建议]` 昂贵操作（JSON 序列化、反射）前先检查日志级别。
```python
❌ LOG.debug("Payload=%s", json.dumps(payload))  # 级别未开启也会执行序列化
✅ if LOG.isEnabledFor(logging.DEBUG): LOG.debug("Payload=%s", json.dumps(payload))
```

---

### 最佳实践

**LOG-PRAC-01** `[强制]` 分布式场景所有日志必须携带 traceId（可获取时）。
违规信号：HTTP handler / RPC handler / 消费函数的 INFO 日志无 `traceId`。
```python
❌ LOG.info("Handling order, orderId=%s", order_id)
✅ LOG.info("Handling order, orderId=%s, traceId=%s", order_id, trace_id)
```

---

**LOG-PRAC-02** `[强制]` ERROR/CRITICAL 必须携带堆栈（`exc_info=True` 或 `LOG.exception()`）。
违规信号：`except` 块中 `LOG.error(...)` 无 `exc_info=True`。
```python
❌ except Exception as e: LOG.error("Failed, error=%s", str(e))
✅ except Exception as e: LOG.error("Failed, error=%s", str(e), exc_info=True); raise
```

---

**LOG-PRAC-03** `[强制]` 关键条件分支必须打印判断条件值和关键参数，以还原执行路径。
违规信号：业务 `if/elif`（状态机/路由/增删改）分支内无日志。
```python
❌ if method == 'alipay': pay_by_alipay(order_id)  # 走哪条路不可见
✅ LOG.info("Payment routed, orderId=%s, method=%s", order_id, method); pay_by_alipay(order_id)
```

---

**LOG-PRAC-04** `[强制]` 临时上线观察日志必须加采样控制并标注 TODO，稳定后立即删除。
违规信号：高并发路径的 WARNING/INFO 无 `should_sample()` 且无 TODO 注释。
```python
❌ LOG.warning("New checkout flow, userId=%s", user_id)  # 高并发下撑爆磁盘
✅ # TODO(临时): 灰度观察，稳定后删除
   if should_sample(rate=0.01): LOG.warning("New checkout flow, userId=%s", user_id)
```

---

**LOG-PRAC-05** `[强制]` 禁止无目的日志（空消息、无参数、重复日志）。
```python
❌ LOG.info("here")
❌ LOG.info("Done")   # 无参数，无法定位
❌ for item in items: LOG.info("Processing")  # 重复且无信息
```

## Code Review Checklist

逐项扫描，违规输出 `[规则ID] <文件>:<行号> — <描述>`。

**日志级别**
- [ ] LOG-LVL-01 — raise/exit 前是否有 CRITICAL
- [ ] LOG-LVL-02 — except 块 raise 前是否有 ERROR
- [ ] LOG-LVL-03 — 业务校验是否误用 ERROR（应 WARNING）
- [ ] LOG-LVL-04 — 有 fallback 的降级场景是否有 WARNING
- [ ] LOG-LVL-05 — 系统/集成故障是否误用 WARNING（应 ERROR）
- [ ] LOG-LVL-06 — 增删改接口入口是否有 INFO 含入参
- [ ] LOG-LVL-07 — 定时任务/队列消费是否有 INFO 含范围/内容
- [ ] LOG-LVL-08 — 循环体（>10次）内是否有 INFO 及以上（应改摘要）
- [ ] LOG-LVL-09 — 高频方法（>100次/秒）是否有 INFO 及以上
- [ ] LOG-LVL-10 — DEBUG 级别是否通过环境变量控制

**消息格式**
- [ ] LOG-FMT-01 — 是否用了 f-string / 拼接 / % 运算符 / extra=（应为 %s 占位符）
- [ ] LOG-FMT-02 — 消息键名是否为 camelCase（禁 snake_case / PascalCase）
- [ ] LOG-FMT-03 — 消息是否包含足够上下文（非单词、含关键参数）
- [ ] LOG-FMT-04 — 是否打印了 None 或无关变量

**参数约定**
- [ ] LOG-PARAM-01 — HTTP 日志含 method/path/statusCode/duration/userId/traceId
- [ ] LOG-PARAM-02 — DB 日志含 table/duration/rowCount/database
- [ ] LOG-PARAM-03 — 文件操作含 filename/path/operation/size/permissions
- [ ] LOG-PARAM-04 — 业务操作含 userId/resourceId/action/result/duration

**敏感信息**
- [ ] LOG-SEC-01~04 — password/token/phone/card 是否脱敏

**性能与最佳实践**
- [ ] LOG-PERF-01 — 昂贵操作前是否检查日志级别
- [ ] LOG-PRAC-01 — 分布式入口日志是否含 traceId
- [ ] LOG-PRAC-02 — ERROR/CRITICAL 是否带 exc_info=True
- [ ] LOG-PRAC-03 — 关键条件分支是否打印条件值和关键参数
- [ ] LOG-PRAC-04 — 临时观察日志是否有采样控制和 TODO 标注
- [ ] LOG-PRAC-05 — 是否存在空日志、无参数日志、重复日志
