# 错误码规范

## 用途

本文档供 AI 执行两类任务：**系统设计**中异常场景中的错误码注册 **代码生成**（定义错误码/异常类前对照规则）、**代码 Review**（逐条扫描输出违规）。

违规报告格式：`[规则ID] <文件>:<行号> — <描述>`

强制等级：`[强制]` 必须遵守 / `[建议]` 推荐遵守

---

## 错误类选择决策树

```
错误是否暴露给前端用户？
  是 → VisibleEx 类
       message_format = _(u"[模块>功能] 原因描述。处理建议")
  否 → 是否用于内部逻辑异常/调试排障？
         是 → InvisibleEx 类 或 具体内部错误类
              message_format = _(u"[模块-资源-操作] 技术细节 %(param)s")
         否 → HTTP 异常类（HTTPBadRequest / HTTPForbidden 等）
              继承 BusinessException，设置 code + title
```

## 规则详情

### 错误码定义

**ERR-DEF-01** `[强制]` 错误码采用 12 位十六进制字符串，格式为 `0xXXXXXXXXXXXX`，不得使用数字或其他格式。
违规信号：`errcode` 为整型、短于 12 位、或不以 `0x` 开头。
```python
❌ errcode = 1001                     # 整型
❌ errcode = "0x1010"                 # 位数不足
✅ errcode = "0x101002000001"
✅ errcode = "0x12008013002d"
```

---

**ERR-DEF-02** `[强制]` 每个模块必须在文件顶部声明 `_MODULE_ERROR_CODE` 范围和 `_SUB_MODULE_ERROR_CODE` 子模块范围，所有错误码必须落在已声明范围内。
违规信号：模块内无错误码范围声明；错误码值超出已声明范围；不同子模块的范围存在重叠。
```python
❌ # 无范围声明，直接定义错误码
   ROUTER_CREATE_ERROR = ("0x101002000001", "路由器创建失败")

✅ _MODULE_ERROR_CODE = ("0x101002000001", "0x101002FFFFFF")
   _SUB_MODULE_ERROR_CODE = {
       "NETWORK": ("0x101002000001", "0x1010020FFFFF"),
       "STORAGE": ("0x101002100000", "0x1010021FFFFF"),
   }
   NETWORK_ROUTER_CREATE_ERROR = ("0x101002000001", "网络路由器创建失败")
```

---

**ERR-DEF-03** `[强制]` 错误码变量命名格式为 `[模块名]_[资源名]_[操作类型]_ERROR`，全部大写，下划线分隔。
违规信号：命名过于笼统（如 `CREATE_ERROR`、`FAIL`）；缺少资源名；后缀不是 `_ERROR`。
```python
❌ CREATE_ERROR = ("0x101002000001", "创建失败")          # 无模块名和资源名
❌ MODULE_A_FAIL = ("0x101002000002", "失败")             # 后缀不规范
✅ NETWORK_ROUTER_CREATE_ERROR = ("0x101002000001", "网络路由器创建失败")
✅ STORAGE_VOLUME_ATTACH_ERROR = ("0x101002000003", "存储卷挂载失败")
✅ USER_AUTH_VALIDATE_ERROR    = ("0x101002000004", "用户认证校验失败")
```

### 错误类

**ERR-CLASS-01** `[强制]` 错误类必须按可见性命名：对用户可见的异常以 `VisibleEx` 开头，内部异常以 `InvisibleEx` 开头或以具体错误类型开头（如 `ClusterReserveResourceError`）；HTTP 异常使用 `HTTP` 前缀。
违规信号：可见错误类名不含 `VisibleEx`；内部调试异常被命名为 `VisibleEx`。
```python
❌ class CreateError(BusinessException): ...          # 无可见性标识
❌ class UserError(BusinessException): ...            # 命名含糊

✅ class VisibleExUserNotFoundError(BusinessException): ...      # 用户可见
✅ class InvisibleExNonVpcRouter(BusinessException): ...         # 内部调试
✅ class ClusterReserveResourceError(BusinessException): ...     # 内部错误
✅ class HTTPBadRequest(BusinessException): ...                  # HTTP 层
```

---

**ERR-CLASS-02** `[强制]` 所有自定义异常必须继承自统一的 `BusinessException` 基类（或其子类），禁止直接继承 Python 内置 `Exception`。
违规信号：`class XxxError(Exception):` 直接继承内置异常；未继承任何业务基类。
```python
❌ class UserCreateError(Exception):
       errcode = "0x101002000001"
       message_format = u"创建失败"

✅ class UserCreateError(BusinessException):
       errcode = "0x101002000001"
       message_format = _(u"用户管理>创建用户失败，原因：%(reason)s。请检查用户信息后重试")
```

---

**ERR-CLASS-03** `[强制]` 每个异常类必须定义 `errcode` 和 `message_format` 属性；HTTP 异常类额外定义 `code`（HTTP 状态码）和 `title`。
违规信号：异常类缺少 `errcode` 或 `message_format`；HTTP 异常类的 `code` 不是有效 HTTP 状态码。
```python
❌ class RouterCreateError(BusinessException):
       message_format = _(u"路由器创建失败")   # 缺少 errcode

❌ class HTTPBadRequest(BusinessException):
       errcode = "0x101002000008"              # 缺少 code 和 title

✅ class VisibleExRouterCreateError(BusinessException):
       errcode = "0x101002000001"
       message_format = _(u"网络管理>路由器创建失败，名称(%(name)s)已存在。请更换名称后重试")

✅ class HTTPBadRequest(BusinessException):
       errcode = "0x12008013002d"
       message_format = _(u"请求参数错误，原因：%(error)s。请检查请求参数格式")
       code = 400
       title = "Bad Request"
```

---

**ERR-CLASS-04** `[强制]` 所有异常类必须集中定义在模块的 `exception.py` 文件中，禁止在业务逻辑文件（views、handlers、utils 等）中散落定义。
违规信号：在非 `exception.py` 文件中出现 `class VisibleEx*` 或 `class InvisibleEx*` 定义；多个 `exception.py` 文件中定义同一模块的异常。
```python
❌ # network/views.py 中
   class VisibleExIPUsedError(BusinessException):
       errcode = "0x101002000010"
       message_format = _(u"IP 已被占用")

✅ # network/exception.py 中集中定义所有该模块异常
   class VisibleExIPUsedError(BusinessException):
       errcode = "0x101002000010"
       message_format = _(u"网络管理>IP 分配失败，IP(%(ip)s)已被占用。请选择其他可用 IP")
```

### 错误信息

**ERR-MSG-01** `[强制]` `message_format` 必须使用 `%(参数名)s` 占位符动态注入上下文，且必须用 `_()` 包装以支持国际化。
违规信号：message_format 为硬编码字符串，无任何参数占位符；未使用 `_()` 包装。
```python
❌ message_format = "用户不存在"                        # 无占位符、无 _()
❌ message_format = _(u"路由器创建失败")                # 无占位符，缺少上下文

✅ message_format = _(u"用户管理>查询用户失败，用户(%(user_name)s)不存在。"
                      u"请确认用户名是否正确或刷新页面后重试")
✅ message_format = _(u"[网络管理-路由器-类型校验失败] "
                      u"路由器(%(router_id)s)不是 VPC 类型，"
                      u"期望：VPC 路由器，实际：%(actual_type)s")
```

---

**ERR-MSG-02** `[强制]` `VisibleEx` 类的 `message_format` 必须包含三要素：**位置信息**（`模块>功能`）+ **错误原因**（含参数化资源标识）+ **处理建议**。
违规信号：message_format 中无 `>` 字符（缺少位置信息）；无参数占位符（原因不具体）；结尾无处理建议或建议过于空泛（仅"请重试"）。
```python
❌ class VisibleExUserNotFoundError(BusinessException):
       errcode = "0x101002000001"
       message_format = _(u"用户不存在，请确认后重试。")   # 无位置，建议空泛

❌ class VisibleExIPConflictError(BusinessException):
       errcode = "0x101002000002"
       message_format = _(u"网络管理>IP 冲突")            # 无原因参数，无建议

✅ class VisibleExUserNotFoundError(BusinessException):
       errcode = "0x101002000001"
       message_format = _(u"用户管理>查询用户失败，用户(%(user_name)s)不存在。"
                          u"请确认用户名是否正确或刷新页面后重试")

✅ class VisibleExIPConflictError(BusinessException):
       errcode = "0x101002000002"
       message_format = _(u"网络管理>弹性 IP 绑定失败，IP(%(ip)s)已被资源(%(resource_id)s)占用。"
                          u"请先解绑该 IP 或选择其他可用 IP")
```

---

**ERR-MSG-03** `[强制]` `InvisibleEx` 类及内部错误类的 `message_format` 必须以 `[模块-资源-操作]` 三层结构开头，并包含至少 2 个技术上下文参数。
违规信号：message_format 不以 `[` 开头；方括号内不含 `-` 分隔的三层结构；参数占位符少于 2 个。
```python
❌ class InvisibleExNonVpcRouter(BusinessException):
       errcode = "0x120080520003"
       message_format = _(u"该路由器%(router_id)s不是 VPC 内路由器，请检查参数!")   # 无层次结构

❌ class InvisibleExDBInsertError(BusinessException):
       errcode = "0x101002000005"
       message_format = _(u"[数据库] 插入失败 %(error)s")   # 层次结构不完整，参数不足

✅ class InvisibleExNonVpcRouter(BusinessException):
       errcode = "0x120080520003"
       message_format = _(u"[网络管理-路由器-类型校验失败] "
                          u"路由器(%(router_id)s)不是 VPC 类型，"
                          u"期望类型：VPC 路由器，实际类型：%(actual_type)s")

✅ class InvisibleExDBInsertError(BusinessException):
       errcode = "0x101002000005"
       message_format = _(u"[用户管理-用户-数据库插入失败] "
                          u"用户(%(username)s)插入失败，表：%(table)s，"
                          u"错误码：%(error_code)s，错误信息：%(error)s")
```

---

**ERR-MSG-04** `[强制]` `message_format` 必须统一使用中文，禁止中英文混用。
违规信号：message_format 中出现英文单词（HTTP 状态码名称、专有名词除外）；中英文混写在同一句话中。
```python
❌ message_format = _(u"User %(user_name)s not found")             # 全英文
❌ message_format = _(u"用户 %(user_name)s not found，请检查")     # 中英混用

✅ message_format = _(u"用户管理>查询用户失败，用户(%(user_name)s)不存在。"
                      u"请确认用户名是否正确或刷新页面后重试")
```

### 模块管理

**ERR-MOD-01** `[强制]` 抛出异常前必须先打印日志，禁止无日志直接 raise。
违规信号：`raise VisibleEx*` / `raise InvisibleEx*` / `raise HTTP*` 前无 `LOG.error` / `LOG.warning`。
```python
❌ if not user:
       raise VisibleExUserNotFoundError(user_name=user_name)   # 无日志

✅ if not user:
       LOG.warning("User not found, userName=%s", user_name)
       raise VisibleExUserNotFoundError(user_name=user_name)

✅ except DBException as e:
       LOG.error("DB insert failed, table=%s, error=%s", table, str(e), exc_info=True)
       raise InvisibleExDBInsertError(table=table, error_code=e.code, error=str(e))
```

---

**ERR-MOD-02** `[强制]` 错误码范围必须集中注册，同一产品线内不允许不同模块使用重叠的错误码范围。
违规信号：两个模块的 `_SUB_MODULE_ERROR_CODE` 范围存在重叠；新增错误码未在范围声明内登记。
```python
❌ # module_a/exception.py
   _SUB_MODULE_ERROR_CODE = {"NET": ("0x101002000001", "0x1010020FFFFF")}
   # module_b/exception.py
   _SUB_MODULE_ERROR_CODE = {"NET": ("0x101002000100", "0x1010020FFFFF")}  # 范围重叠

✅ # module_a 分配 000001~0FFFFF，module_b 分配 100000~1FFFFF，不重叠
   # 新增错误码 0x101002000050 落在 000001~0FFFFF 范围内，合法
```

---

**ERR-MOD-03** `[建议]` 资源名与模块名使用统一的映射表，避免同一资源在不同错误码中使用不同名称（如 Router/路由器/router 混用）；映射值若用于拼接用户可见 message_format，必须用 `_()` 包裹。
```python
# 推荐维护团队级资源-模块映射（值用于拼接 message_format 时必须 i18n 包裹）
RESOURCE_MODULE_MAP = {
    "User": _(u"用户管理"), "Project": _(u"租户管理"),
    "Router": _(u"网络管理"), "FloatingIP": _(u"网络管理"), "Subnet": _(u"网络管理"),
    "Server": _(u"计算管理"), "Instance": _(u"计算管理"),
    "Volume": _(u"存储管理"), "Disk": _(u"存储管理"),
    "Cluster": _(u"集群管理"), "Database": _(u"数据库"), "Metric": _(u"监控模块"),
}
OPERATION_MAP = {
    "Create": _(u"创建"), "Delete": _(u"删除"), "Update": _(u"更新"),
    "NotFound": _(u"查询/未找到"), "Validate": _(u"校验"), "Connect": _(u"连接"),
    "Attach": _(u"挂载"), "Detach": _(u"卸载"), "Bind": _(u"绑定"),
}
```

## Go 错误处理（internal/errcode/errors.go）

> Go 没有 raise/except 机制，错误以返回值传递。本节规范基于 CMP/Core/Srv 下各服务的真实实现提炼。

### 错误结构决策树

```
错误是否暴露给前端用户？
  是 → Visible 前缀变量
       Message  = i18n.Sprintf("[操作]失败/错误，[失败原因]，[解决方法]")  ← 三段式全写入 Message
       Solution = i18n.Sprintf("[解决方法]")                               ← 复制 Message 中的解决方法即可
  否 → 是否为框架/通用错误（参数非法、服务不可用）？
         是 → 无前缀变量，Message = "English: [op] failed: %v"（不国际化）
         否 → 无前缀或 Inter 前缀变量，Message = "English: [op] failed: %v"（英文）
动态参数注入 → 调用侧使用 .FmtErrCodeMessage(args...)
下游错误链    → 调用侧使用 .AppendErrCode(downstreamErr)
```

### ErrorCode 结构体（标准定义）

```go
// internal/errcode/errcode.go（每个服务均有此文件，结构一致）
type ErrorCode struct {
    ErrCode      string   `json:"errcode"`           // 12 位十六进制
    Message      string   `json:"message,omitempty"` // 错误消息
    Solution     string   `json:"solution,omitempty"`// 处理建议（独立字段）
    Code         string   `json:"code,omitempty"`    // HTTP 状态码字符串，如 "400"
    ErrCodeStack []string `json:"errcode_stack,omitempty"` // 下游错误链
}

func (e ErrorCode) Error() string { /* JSON 序列化 */ }

// FmtErrCodeMessage 注入运行时动态参数（Message 中使用 %v/%s 占位）
func (e ErrorCode) FmtErrCodeMessage(a ...interface{}) ErrorCode

// AppendErrCode 将下游 ErrorCode 追加到 ErrCodeStack
func (e ErrorCode) AppendErrCode(err ErrorCode) ErrorCode

// Error2ErrCode 从 err.Error() 反序列化回 ErrorCode
func Error2ErrCode(err error) ErrorCode
```

### 错误定义

**ERR-GO-DEF-01** `[强制]` Go 错误变量必须集中定义在 `internal/errcode/errors.go`，禁止在 handler、service、dao 等业务文件中散落定义。
违规信号：业务文件中出现包级 `var Xxx = &ErrorCode{...}` 或 `errors.New()` 定义可复用的错误变量。
```go
❌ // internal/api/handler/router_handler.go
   var routerNotFound = &errcode.ErrorCode{       // 散落在业务文件
       ErrCode: "0x101002000001",
       Message: i18n.Sprintf("路由器不存在"),      // 无操作描述，无原因，缺 Solution
   }

✅ // internal/errcode/errors.go
   var VisibleRouterNotFoundError = &ErrorCode{
       ErrCode:  "0x101002000001",
       Message:  i18n.Sprintf("查询路由器失败，路由器不存在，请确认路由器 ID 是否正确"),
       Solution: i18n.Sprintf("请确认路由器 ID 是否正确后重试"),
       Code:     "400",
   }
```

---

**ERR-GO-DEF-02** `[强制]` `ErrorCode` 结构体必须包含 `ErrCode`（12 位十六进制）、`Message`、`Solution`、`Code`（HTTP 状态码）四个字段；其中 `Message` 和 `Solution` 对用户可见的必须用 `i18n.Sprintf()` 包裹，内部错误用英文字面量。
违规信号：缺少 `ErrCode` 字段；用户可见 `Message`/`Solution` 未用 `i18n.Sprintf()` 包裹；内部错误用了 `i18n.Sprintf()`。
```go
❌ var TemplateDeleteError = &ErrorCode{
       Message: "该模版存在进行中的任务，暂不支持删除",  // 未用 i18n.Sprintf；缺 ErrCode；缺 Solution
   }

❌ var InvalidRequestError = &ErrorCode{
       ErrCode:  "0x010070020001",
       Message:  i18n.Sprintf("Invalid request: %v"),  // 内部错误不需要 i18n
       Code:     "400",
   }

✅ // 用户可见错误：Message = 操作+原因+解决方法；Solution 与 Message 中解决方法一致
   var VisibleTemplateInProgressError = &ErrorCode{
       ErrCode:  "0x0101B0010007",
       Message:  i18n.Sprintf("删除模版失败，该模版存在进行中的任务，请等待任务完成后重试"),
       Solution: i18n.Sprintf("请等待任务完成后重试"),
       Code:     "400",
   }

✅ // 内部/框架错误：英文字面量，无需 i18n
   var InvalidRequest = &ErrorCode{
       ErrCode:  "0x010070020001",
       Message:  "Invalid request: %v",
       Solution: "Please check your request",
       Code:     "400",
   }
```

---

**ERR-GO-DEF-03** `[强制]` 用户可见错误变量以 `Visible` 前缀命名；框架级/内部错误无前缀（如 `InvalidRequest`、`InternalServerError`）；跨服务引用的外部错误码需加注释说明来源。
违规信号：用户可见错误不含 `Visible` 前缀导致无法区分；直接在调用侧 `return &ErrorCode{...}` 而不定义命名变量。
```go
❌ var RouterNotFound = &ErrorCode{           // 用户可见但无 Visible 前缀
       Message: i18n.Sprintf("路由器不存在"), // 缺操作描述，缺 Solution
   }
❌ // 调用侧临时构造
   return errcode.ErrorCode{ErrCode: "0x...", Message: i18n.Sprintf("...")}

✅ // 用户可见错误：Visible 前缀，Message=操作+原因+解决方法
   var VisibleCloseAggregationEventError = &ErrorCode{
       ErrCode:  "0x010070020008",
       Message:  i18n.Sprintf("关闭聚合事件失败，聚合事件状态异常，请检查告警服务和数据库是否正常后重试"),
       Solution: i18n.Sprintf("请检查告警服务和数据库是否正常后重试"),
       Code:     "400",
   }

✅ // 框架/内部错误：无前缀，英文
   var InternalServerError = &ErrorCode{
       ErrCode:  "0x0100d0010002",
       Message:  "Internal server error: %v",
       Solution: "Please contact admin",
       Code:     "500",
   }

✅ // 引用外部服务错误码（加注释说明来源）
   // VisibleExInvalidReceiverError 此错误码定义在 sf-notify，仅作引用，不可直接返回
   var VisibleExInvalidReceiverError = &ErrorCode{
       ErrCode:  "0x010250010001",
       Message:  i18n.Sprintf("发送通知失败，通知接收人无效，请检查通知接收人配置是否正确"),
       Solution: i18n.Sprintf("请检查通知接收人配置是否正确"),
       Code:     "400",
   }
```

### 错误信息（Go）

**ERR-GO-MSG-01** `[强制]` 用户可见错误（`Visible` 前缀）的 `Message` 必须包含三段式：**操作描述** + **失败原因** + **解决方法**；`Solution` 复制 Message 中的解决方法即可（与 Message 保持一致）。两者均须用 `i18n.Sprintf()` 包裹。
违规信号：`Message` 仅说"失败"但无具体原因或解决方法；`Message` 与 `Solution` 完全相同；未使用 `i18n.Sprintf()`。

`Message` 三段式格式：`i18n.Sprintf("[操作]失败/错误，[失败原因]，[解决方法]")`

```go
❌ var VisibleAcquireLockFailed = &ErrorCode{
       ErrCode:  "0x0101B0010008",
       Message:  i18n.Sprintf("操作失败"),                      // 无操作描述，无原因，无解决方法
```

### 模块管理（Go）

**ERR-GO-MOD-01** `[强制]` 返回 `ErrorCode` 前必须先打日志；含动态参数时通过 `.FmtErrCodeMessage(args...)` 注入，禁止在变量定义时硬编码动态值。
违规信号：`return errcode.XxxError` 前无日志；直接修改变量的 `Message` 而不用 `FmtErrCodeMessage`。
```go
❌ // 无日志直接返回
   if err != nil {
       return errcode.InvalidRequest
   }

❌ // 在变量定义时硬编码动态值（变量被污染）
   errcode.InvalidRequest.Message = fmt.Sprintf("Invalid request: %v", err)
   return errcode.InvalidRequest

✅ if err != nil {
       slogger.WithContext(ctx).Errorf("parse request failed, err=%v", err)
       return errcode.InvalidRequest.FmtErrCodeMessage(err)
   }
```

---

**ERR-GO-MOD-02** `[强制]` 调用下游服务返回的 `ErrorCode` 需追加到 `ErrCodeStack` 中，禁止直接丢弃下游错误码或直接透传。
违规信号：`return downstreamErr` 直接透传下游错误；`return errcode.InternalServerError` 丢弃了下游错误码信息。
```go
❌ if downstreamErr := callService(); downstreamErr != nil {
       return errcode.InternalServerError   // 丢弃下游错误码
   }

❌ if downstreamErr := callService(); downstreamErr != nil {
       return downstreamErr   // 直接透传，无本层语义
   }

✅ if downstreamErr := callService(); downstreamErr != nil {
       slogger.WithContext(ctx).Errorf("call downstream failed, err=%v", downstreamErr)
       downEC := errcode.Error2ErrCode(downstreamErr)
       return errcode.InternalServerError.AppendErrCode(downEC)
   }
```

---

**ERR-GO-MOD-03** `[强制]` 禁止忽略 error 返回值；禁止只打日志不 return（既不 return 也不中止执行）。
违规信号：`someFunc()` 未捕获 error；`if err != nil { log.Error(...) }` 后继续执行后续逻辑。
```go
❌ db.Insert(record)   // 完全忽略 error

❌ if err := db.Insert(record); err != nil {
       slogger.WithContext(ctx).Errorf("insert failed, err=%v", err)
       // 未 return，后续逻辑继续执行
   }

✅ if err := db.Insert(record); err != nil {
       slogger.WithContext(ctx).Errorf("db insert failed, table=reports, err=%v", err)
       return errcode.InternalServerError.FmtErrCodeMessage(err)
   }
```

## Code Review Checklist

逐项扫描，违规输出 `[规则ID] <文件>:<行号> — <描述>`。

**错误码定义**
- [ ] ERR-DEF-01 — errcode 是否为 12 位十六进制字符串（`0x` + 12 位）
- [ ] ERR-DEF-02 — 模块是否声明 `_MODULE_ERROR_CODE` 和 `_SUB_MODULE_ERROR_CODE`，且无范围重叠
- [ ] ERR-DEF-03 — 变量命名是否符合 `[模块]_[资源]_[操作]_ERROR` 格式

**错误类**
- [ ] ERR-CLASS-01 — 可见错误是否以 `VisibleEx` 开头，内部错误是否以 `InvisibleEx` 或具体类型开头
- [ ] ERR-CLASS-02 — 是否继承 `BusinessException`（禁止直接继承 `Exception`）
- [ ] ERR-CLASS-03 — 是否定义 `errcode` 和 `message_format`；HTTP 异常是否有 `code` 和 `title`
- [ ] ERR-CLASS-04 — 异常类是否集中定义在 `exception.py`（禁止在 views/handlers 中定义）

**错误信息**
- [ ] ERR-MSG-01 — message_format 是否含 `%(param)s` 占位符且用 `_()` 包装
- [ ] ERR-MSG-02 — VisibleEx 的 message_format 是否含位置（`>`）+ 原因（占位符）+ 处理建议
- [ ] ERR-MSG-03 — InvisibleEx 的 message_format 是否以 `[模块-资源-操作]` 开头且含 ≥2 个参数
- [ ] ERR-MSG-04 — message_format 是否统一中文（禁止中英混用）

**模块管理（Python）**
- [ ] ERR-MOD-01 — raise 前是否有 LOG.warning/LOG.error 日志
- [ ] ERR-MOD-02 — 错误码是否集中注册且无跨模块范围重叠
- [ ] ERR-MOD-03 — 资源名/模块名是否使用统一映射（建议）

**Go 错误定义（internal/errcode/errors.go）**
- [ ] ERR-GO-DEF-01 — 错误变量是否集中定义在 `internal/errcode/errors.go`（禁止业务文件中散落定义）
- [ ] ERR-GO-DEF-02 — `ErrorCode` 是否包含 `ErrCode`/`Message`/`Solution`/`Code` 四字段；用户可见字段是否用 `i18n.Sprintf()` 包裹
- [ ] ERR-GO-DEF-03 — 用户可见错误是否以 `Visible` 前缀命名；外部引用错误码是否有注释说明来源

**Go 错误信息**
- [ ] ERR-GO-MSG-01 — `Visible` 错误的 `Message` 是否描述具体原因，`Solution` 是否给出处理建议，二者是否分别用 `i18n.Sprintf()` 包裹
- [ ] ERR-GO-MSG-02 — 用户可见字段（`i18n.Sprintf`）是否为纯中文；内部错误是否为英文（禁止中英混用）

**Go 模块管理**
- [ ] ERR-GO-MOD-01 — return ErrorCode 前是否有日志；动态参数是否通过 `.FmtErrCodeMessage()` 注入（禁止直接修改变量字段）
- [ ] ERR-GO-MOD-02 — 下游 ErrorCode 是否通过 `.AppendErrCode()` 追加到 ErrCodeStack（禁止丢弃或直接透传）
- [ ] ERR-GO-MOD-03 — 是否存在忽略 error 返回值或只打日志不 return 的情况（禁止）
