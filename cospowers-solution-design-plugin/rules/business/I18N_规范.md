## Python 2 国际化规范

# Python 2 国际化编码规范 (I18N)

## 用途

本文档供 AI 执行两类任务：**代码生成**（写字符串前对照规则）、**代码 Review**（逐条扫描输出违规）。

违规报告格式：`[规则ID] <文件>:<行号> — <描述>`

强制等级：`[强制]` 必须遵守 / `[建议]` 推荐遵守

---

## 字符串分类决策树

```
字符串用途？
  直接展示给最终用户（UI文本、错误消息、提示信息、审计日志）
    → 用户可见字符串 → 必须用 _() 包裹，且含中文时必须加 u 前缀
  存储在数据库的业务内容（产品分类、系统预设选项）
    → 业务数据 → 视情况国际化（存储翻译键，显示时解析）
  仅供开发者使用（调试日志、配置键、常量、代码注释）
    → 技术性字符串 → 可以硬编码
```

---

## 规则详情

### 导入规范

**I18N-PY2-IMPORT-01** `[建议]` 使用 `gettext` 标准库并导入 `from gettext import gettext as _`，禁止自定义别名。

违规信号：使用其他别名如 `from gettext import gettext as tr`；使用第三方 i18n 库而不使用标准 `gettext`。

```python
❌ from gettext import gettext as tr              # 非标准别名
❌ import some_i18n_lib as _                       # 非标准库

✅ from gettext import gettext as _
✅ from gettext import ngettext
```

---

### 字符串分类

**I18N-PY2-CLASS-01** `[强制]` 所有字符串必须按三类处理：用户可见字符串必须 `_()` 包裹；技术性字符串可以硬编码；业务数据视情况国际化。

违规信号：用户可见字符串被当作技术性字符串处理（未包裹 `_()`）；技术性字符串被过度国际化；业务数据直接存储中文而未存储翻译键。

```python
❌ return {"message": "操作成功"}                           # 用户可见，未国际化
❌ logger.debug(_(u"调试信息"))                              # 技术性，过度国际化
❌ category = {"name": u"云计算"}                            # 业务数据，硬编码中文

✅ return {"message": _(u"操作成功")}                       # 用户可见，已国际化
✅ logger.debug("Debug info")                               # 技术性，硬编码
✅ category = {"name": "category.cloud_computing"}          # 业务数据，存储翻译键
   display = _(category["name"])
```

---

### 占位符命名

**I18N-PY2-PARAM-01** `[建议]` 占位符使用语义化命名（如 `{username}`、`{fileSize}`），避免使用序号（`{0}`、`{1}`）。

违规信号：大量使用 `{0}`、`{1}` 等序号占位符，翻译人员无法判断上下文。

```python
❌ _(u"用户 {0} 在 {1} 登录").format(username, timestamp)      # 序号占位符

✅ _(u"用户 {username} 在 {time} 登录").format(username=username, time=timestamp)
```

---

### 字符串包裹

**I18N-PY2-WRAP-01** `[强制]` 所有用户可见字符串必须使用 `_()` 包裹，含中文时必须加 `u` 前缀，禁止硬编码。

违规信号：裸中文字符串；`_()` 内中文未加 `u` 前缀。

```python
❌ print("操作成功")
❌ print(_("操作成功"))                    # 中文未加 u 前缀
❌ raise ValueError("用户名不能为空")
❌ return {"error": "权限不足"}

✅ print(_(u"操作成功"))
✅ raise ValueError(_(u"用户名不能为空"))
✅ return {"error": _(u"权限不足")}
```

---

### 格式化方式

**I18N-PY2-FMT-01** `[强制]` 禁止字符串拼接（`+`），必须使用 `.format()` 或 `%` 运算符参数化。

违规信号：使用 `+` 拼接含中文的字符串；直接拼接变量和字面量。

```python
❌ message = u"用户 " + username + u" 登录成功"
❌ error = u"文件大小不能超过 " + str(max_size) + u"MB"

✅ message = _(u"用户 {username} 登录成功").format(username=username)
✅ error = _(u"文件大小不能超过 {size}MB").format(size=max_size)

# 次选：% 格式化（兼容旧代码）
✅ message = _(u"用户 %s 登录成功") % username
```

---

**I18N-PY2-FMT-02** `[强制]` 含中文的格式化字符串必须先 `_()` 包裹再加 `.format()` / `%`，禁止先格式化后包裹。

违规信号：先 `.format()` 或 `%` 格式化后再套 `_()`。

```python
❌ message = _(u"用户 {username} 登录成功".format(username=username))   # 先 format 后 _()
❌ message = _((u"用户 %s 登录成功") % username)                        # 先 % 后 _()

✅ message = _(u"用户 {username} 登录成功").format(username=username)
✅ message = _(u"用户 %s 登录成功") % username
```

---

**I18N-PY2-FMT-03** `[强制]` 生成包含多项动态字段的消息时，必须先对各项国际化，再参数化合成最终消息；禁止先拼接合成后再包裹 `_()`。

违规信号：先拼接多个字符串片段再套 `_()`；`join` 含中文列表后套 `_()`。

```python
❌ fields = [u"用户名", u"邮箱", u"电话"]
   message = _(u"以下字段不能为空：" + u", ".join(fields))              # 先拼接后 _()

❌ fields_str = u", ".join([_(u"用户名"), _(u"邮箱"), _(u"电话")])
   message = _(u"以下字段不能为空：{fields}".format(fields=fields_str))   # 先 format 后 _()

✅ fields = [_(u"用户名"), _(u"邮箱"), _(u"电话")]
   fields_str = u", ".join(fields)
   message = _(u"以下字段不能为空：{fields}").format(fields=fields_str)
```

---

### 复数处理

**I18N-PY2-PLURAL-01** `[建议]` 涉及数量的字符串使用 `gettext.ngettext()` 处理单复数形式。

```python
from gettext import ngettext

❌ message = _(u"{count} 个文件").format(count=count) if count > 1 else _(u"1 个文件")

✅ message = ngettext(
    u"{count} 个文件",
    u"{count} 个文件",
    count
).format(count=count)
```

---

### 例外情况

**I18N-PY2-EXC-01** `[强制]` 以下技术性字符串可以硬编码，无需 `_()`：

- 调试日志/开发日志
- 配置键名、字典键
- 代码注释和文档字符串
- 技术性常量（API 版本号、正则表达式）
- 日志中的字段名（`userId=`、`table=`）

```python
❌ API_VERSION = _(u"v1.0")
❌ config = {_(u"timeout"): 30}
❌ logger.debug(_(u"处理用户记录: %s"), user_id)

✅ API_VERSION = "v1.0"
✅ config = {"timeout": 30}
✅ logger.debug("Processing user record: %s", user_id)
```

---

### 日志国际化

**I18N-PY2-LOG-01** `[强制]` 审计日志/用户日志必须国际化；调试日志硬编码英文。

```python
❌ audit_log.info(u"用户 admin 执行了删除操作")
❌ logger.debug(_(u"调试信息: %s"), data)

✅ audit_log.info(_(u"用户 {user} 执行了操作 {action}").format(user=username, action=action))
✅ logger.debug("Debug info: %s", data)
```

---

### 异常消息国际化

**I18N-PY2-ERR-00** `[强制]` 所有可能展示给用户的异常消息（`raise` / `return` 的错误信息）必须国际化。

违规信号：`raise` 或 `return` 的错误消息为裸中文字符串；异常类 `message_format` 未用 `_()` 包裹。

```python
❌ raise ValueError("参数无效")
❌ raise BusinessException(message="用户不存在")
❌ return Response.error("权限不足")

✅ raise ValueError(_(u"参数无效"))
✅ raise BusinessException(message=_(u"用户不存在，请确认后重试"))
✅ return Response.error(_(u"权限不足，请联系管理员"))
```

---

### 第三方错误处理

**I18N-PY2-ERR-01** `[强制]` 第三方库错误不得直接返回，必须包装为国际化消息。

```python
❌ except ExternalAPIError as e:
       return {"error": str(e)}

✅ except ExternalAPIError as e:
       logger.error("External API error: %s", str(e))
       return {"error": _(u"外部服务调用失败，请稍后重试")}
```

---

### 业务数据

**I18N-PY2-DB-01** `[建议]` 数据库中的系统预设选项/产品分类需国际化（存储翻译键，显示时解析）；用户输入的数据保持原样。

违规信号：数据库中直接存储中文描述；系统预设选项未存储翻译键。

```python
❌ category = {"name": u"云计算", "desc": u"云服务器产品"}    # 硬编码中文存入数据库

✅ category = {"name": "category.cloud_computing", "desc": "category.cloud_computing.desc"}
   display_name = _(category["name"])                              # 显示时解析
```

违规信号：数据库中直接存储中文描述；系统预设选项未存储翻译键。

```python
❌ category = {"name": "云计算", "desc": "云服务器产品"}    # 硬编码中文存入数据库

✅ category = {"name": "category.cloud_computing", "desc": "category.cloud_computing.desc"}
   display_name = _(category["name"])
```

## Go 国际化规范

# Go 国际化编码规范 (I18N)

## 用途

本文档供 AI 执行两类任务：**代码生成**（写字符串前对照规则）、**代码 Review**（逐条扫描输出违规）。

违规报告格式：`[规则ID] <文件>:<行号> — <描述>`

强制等级：`[强制]` 必须遵守 / `[建议]` 推荐遵守

---

## 字符串分类决策树

```
字符串用途？
  直接展示给最终用户（UI文本、错误消息、提示信息、审计日志）
    → 用户可见字符串 → 必须用 i18n.I() 或 i18n.Sprintf() 包裹
  存储在数据库的业务内容（产品分类、系统预设选项）
    → 业务数据 → 视情况国际化（存储翻译键，显示时解析）
  仅供开发者使用（调试日志、配置键、常量、代码注释）
    → 技术性字符串 → 可以硬编码
```

```
需要参数注入？
  是 → i18n.Sprintf("...", args...)
  否 → i18n.I("...")
```

---

## 规则详情

### 字符串分类

**I18N-GO-CLASS-01** `[强制]` 所有字符串必须按三类处理：用户可见字符串必须 `i18n.I()` / `i18n.Sprintf()` 包裹；技术性字符串可以硬编码；业务数据视情况国际化（存储翻译键，显示时解析）。

违规信号：用户可见字符串未包裹 i18n；技术性字符串被过度国际化；系统预设数据直接存储中文。

```go
❌ return map[string]interface{}{"message": "操作成功"}   // 用户可见，未国际化
❌ logger.Debug(i18n.Sprintf("调试信息: %s", data))      // 技术性，过度国际化
❌ category := map[string]string{"name": "云计算"}        // 业务数据，硬编码中文

✅ return map[string]interface{}{"message": i18n.I("操作成功")}
✅ logger.Debugf("Debug info: %s", data)                 // 技术性，硬编码英文
✅ category := map[string]string{"name": "category.cloud_computing"}
   display := i18n.I(category["name"])
```

---

### 占位符命名

**I18N-GO-PARAM-01** `[建议]` 格式化字符串中的占位符参数应语义化命名（通过变量名传递语义），避免无意义参数。

违规信号：`i18n.Sprintf("%s %s %s", a, b, c)` 参数过多且无注释说明；消息字符串本身语义模糊。

```go
❌ message := i18n.Sprintf("%s %s %s", a, b, c)           // 参数无语义

✅ message := i18n.Sprintf("用户 %s 在 %s 登录", username, timestamp)
✅ message := i18n.Sprintf("文件 %s 大小 %d MB", filename, size)
```

---

### 仅通过国际化函数返回用户消息

**I18N-GO-RETURN-01** `[强制]` 所有返回给前端/用户的消息（HTTP 响应、错误返回值、提示文本）必须通过 `i18n.I()` 或 `i18n.Sprintf()` 生成，禁止直接返回硬编码字符串。

违规信号：HTTP handler 中直接写入裸中文字符串；返回的 error 或 response 未经过 i18n 处理。

```go
❌ c.JSON(200, map[string]interface{}{"message": "操作成功"})
❌ return errors.New("参数错误")

✅ c.JSON(200, map[string]interface{}{"message": i18n.I("操作成功")})
✅ return errors.New(i18n.I("参数错误，请检查输入"))
```

---

### 字符串包裹

**I18N-GO-WRAP-01** `[强制]` 所有用户可见字符串必须使用 `i18n.I()`（无参数）或 `i18n.Sprintf()`（有参数）包裹，禁止硬编码。

违规信号：裸中文字符串作为用户消息；裸英文字符串作为用户可见内容。

```go
❌ fmt.Println("操作成功")
❌ return errors.New("用户名不能为空")
❌ log.Printf("权限不足")
❌ message := "用户登录成功"

✅ fmt.Println(i18n.I("操作成功"))
✅ return errors.New(i18n.I("用户名不能为空"))
✅ log.Printf(i18n.I("权限不足"))
✅ message := i18n.I("用户登录成功")
```

---

### 格式化方式

**I18N-GO-FMT-01** `[强制]` 禁止字符串拼接（`+`），必须使用 `i18n.Sprintf()` 参数化。

违规信号：使用 `+` 拼接含中文的字符串；`fmt.Sprintf` 直接拼接含中文的字符串而不经 `i18n.Sprintf`。

```go
❌ message := "用户 " + username + " 登录成功"
❌ error := "文件大小不能超过 " + strconv.Itoa(maxSize) + "MB"
❌ message := fmt.Sprintf("用户 %s 登录成功", username)   // 直接 fmt.Sprintf 含中文

✅ message := i18n.Sprintf("用户 %s 登录成功", username)
✅ error := i18n.Sprintf("文件大小不能超过 %dMB", maxSize)
```

---

### 例外情况

**I18N-GO-EXC-01** `[强制]` 以下技术性字符串可以硬编码，无需 `i18n.I()` / `i18n.Sprintf()`：

- 调试日志/开发日志
- 配置键名、常量
- 代码注释
- 技术性常量（API 版本号、正则表达式）
- 日志中的字段名（`userId=`、`table=`）

```go
❌ const apiVersion = i18n.I("v1.0")
❌ logger.Debug(i18n.Sprintf("处理用户: %s", userID))

✅ const apiVersion = "v1.0"
✅ logger.Debugf("Processing user: %s", userID)
```

---

### 日志国际化

**I18N-GO-LOG-01** `[强制]` 审计日志/用户日志必须国际化；调试日志硬编码英文。

```go
❌ auditLog.Info("用户 admin 执行了删除操作")
❌ logger.Debug(i18n.Sprintf("调试信息: %s"), data)

✅ auditLog.Info(i18n.Sprintf("用户 %s 执行了操作 %s", username, action))
✅ logger.Debugf("Debug info: %s", data)
```

---

### 第三方错误处理

**I18N-GO-ERR-01** `[强制]` 第三方库错误不得直接返回，必须包装为国际化消息。

```go
❌ if err := callExternal(); err != nil {
       return err
   }

✅ if err := callExternal(); err != nil {
       logger.Errorf("external call failed: %v", err)
       return errors.New(i18n.I("外部服务调用失败，请稍后重试"))
   }
```

---

### 错误消息国际化

**I18N-GO-ERR-00** `[强制]` 所有可能展示给用户的错误消息（`return error`、`c.JSON` 错误响应、`fmt.Errorf` 含中文）必须使用 `i18n.I()` 或 `i18n.Sprintf()` 包裹。

违规信号：`return errors.New("...")` 中裸中文；`fmt.Errorf("...")` 中裸中文；HTTP 错误响应直接写中文。

```go
❌ return errors.New("用户名不能为空")
❌ return fmt.Errorf("创建失败: %v", err)
❌ c.JSON(400, map[string]interface{}{"error": "参数错误"})

✅ return errors.New(i18n.I("用户名不能为空"))
✅ return fmt.Errorf(i18n.Sprintf("创建用户失败，原因: %v", err))
✅ c.JSON(400, map[string]interface{}{"error": i18n.I("参数错误，请检查输入")})
```

---

**I18N-GO-ERR-02** `[强制]` 构造格式化错误消息时必须使用 `i18n.Sprintf()` 参数化，禁止字符串拼接或裸 `fmt.Errorf` 含中文。

违规信号：`fmt.Errorf("中文: %v", err)` 中裸中文；`errors.New("中文" + var)` 拼接。

```go
❌ return fmt.Errorf("用户名长度不能少于 %d 个字符", 3)           // 裸 fmt.Errorf 含中文
❌ return errors.New("用户名" + name + "已存在")                  // 字符串拼接

✅ return fmt.Errorf(i18n.Sprintf("用户名长度不能少于 %d 个字符", 3))
✅ return errors.New(i18n.Sprintf("用户名 %s 已存在", name))
```

---

**I18N-GO-ERR-03** `[强制]` 自定义错误类型的 `Error()` 方法返回值必须使用 `i18n.Sprintf()` 或 `i18n.I()` 格式化。

违规信号：自定义 `Error() string` 返回裸中文字符串；返回硬编码拼接字符串。

```go
❌ type ValidationError struct {
       Field   string
       Message string
   }
   func (e *ValidationError) Error() string {
       return e.Field + ": " + e.Message   // 拼接，未国际化
   }

✅ type ValidationError struct {
       Field   string
       Message string
   }
   func (e *ValidationError) Error() string {
       return i18n.Sprintf("%s: %s", e.Field, e.Message)
   }
```

---

### 复数处理

**I18N-GO-PLURAL-01** `[建议]` 涉及数量的字符串使用 `i18n.Plural()` 处理复数形式，禁止用条件分支拼接。

违规信号：`if count > 1` 分支拼接单复数字符串；直接用 `fmt.Sprintf("%d 个文件", count)` 而不区分单复数。

```go
❌ if count == 1 {
       message = i18n.I("1 个文件")
   } else {
       message = i18n.Sprintf("%d 个文件", count)
   }

✅ message = i18n.Plural(count, "file.count.singular", "file.count.plural", count)
```

### 工具集成

**I18N-GO-TOOL-01** `[建议]` 在 pre-commit 或 CI 中集成 Go i18n 扫描器，配置需检查的 i18n 函数列表。

```bash
# pre-commit hook 示例
 tools/go_i18n_scanner/go_i18n_scanner \
   --i18n-funcs "i18n.I,i18n.Sprintf,i18n.Tf" \
   --exclude-test \
   .

# CI 集成（GitLab CI）
 i18n-check:
   stage: lint
   script:
     - tools/go_i18n_scanner/go_i18n_scanner \
         --i18n-funcs "i18n.I,i18n.Sprintf,i18n.Tf" \
         --exclude-test .
```

### 代码审查

**I18N-GO-REVIEW-01** `[强制]` 代码审查时必须检查：所有用户可见字符串是否使用了 `i18n.I()` 或 `i18n.Sprintf()`；调试日志是否保持硬编码英文。

审查要点：
- 搜索文件中的裸中文字符串（`"[^"]*[一-龥][^"]*"`）确认是否为用户可见
- 确认 `errors.New()`、`fmt.Errorf()`、`c.JSON()` 中的中文均经过 i18n 包裹
- 确认 `logger.Debugf()`、`logger.Infof()` 中的日志为英文硬编码

|------|------|------|
| 简单字符串 | `i18n.I(string)` | `i18n.I("操作成功")` |
