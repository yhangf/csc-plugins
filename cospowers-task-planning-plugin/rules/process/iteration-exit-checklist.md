### 1.1 所有用例执行完成

- **质量要求**：

- **平台使用要求**：所有的案例都需上TP进行管理和记录执行结果
- **交付物**：TP上的案例及其执行结果
- **责任人**：TSE

### 1.2 需求验收评审通过

- **标准分类**：质量标准
- **达成标准**：需求已合入到演示/验收环境中，经过验收评审会，且验收问题均已处置
- **裁剪原则**：不可裁剪
- **质量要求**：有需求/特性验收纪要（邮件），且问题已录入TD
- **交付物**：需求/特性验收纪要、TD问题记录
- **责任人**：TL

### 1.3 所有TD问题已闭环

- **标准分类**：质量标准
- **达成标准**：所有TD问题已闭环（closed、halt-2、wontfix），挂起的TD需满足外部确认要求（具体可见 项目缺陷挂起流程）
- **裁剪原则**：不可裁剪
- **质量要求**：
  - TD需可按迭代做区分，原则上最终只存在3种状态（解决/非问题/重复问题-关闭、halt-2、wontfix），如有例外情况向研发主管邮件申请批准
  - halt-2、wontfix的TD已按缺陷挂起流的要求进行确认
- **平台使用要求**：所有的问题都需上TD跟踪
- **责任人**：TL

### 1.4 安全质量保障评审检视

- **标准分类**：质量标准
- **达成标准**：迭代准出安全质量保障评审检视已完成
- **裁剪原则**：不可裁剪
- **质量要求**：
  - 迭代准入识别出的安全需求项100%落地和闭环
  - 登录前暴露面有无新增或改动（端口/接口/参数/header等），必须满足：
    1. ngswaf、可信token（非单机可不选）规则要覆盖防护新增或改动的暴露面
    2. 登录前参数按要求严格校验类型长度字符
    3. 登录前C/C++二进制程序或共享库完成安全编译（共享库，PIE选项不要求）
    4. 新增或改动的登录前暴露面更新至在线文档上
  - 涉及php/python/java/lua/perl自研后端业务代码，对源码进行扫描，有不通过项准出前闭环解决
  - 代码中涉及登录前C/C++二进制程序或共享库，对指定文件或路径进行扫描，有不通过项准出前闭环解决
  - 对模块完成安全审计（代码审计、渗透测试）发现的问题，在迭代准出前都完成修复和闭环确认
- **平台使用要求**：IPD平台跟踪
- **交付物**：验收结论
- **责任人**：安全经理

### 1.5 跨部门协作需求代码合入

### 1.7 代码扫描

- **标准分类**：质量标准
- **达成标准**：代码扫描通过
- **裁剪原则**：未涉及语言类型模块可裁剪
- **质量要求**：
  - C/C++/Golang适用：消除所有编译警告。编译过程必须开启高级别警告选项
    - VC：必须开启3级以上（/W3）告警（库的头文件例外）
    - gcc/g++/gccgo：-Wall -Wextra -Wconversion -Wshadow -Werror
  - 修改的老模块包括在内，如有特殊情况邮件向质量主管申请裁剪
  - 开源模块和历史模块修改：最低要求，千流上的编译告警通过截图
- **交付物**：千流上的编译告警通过截图，编译截图证明（说明已启用高级别告警且已消除所有编译告警）
- **责任人**：TL
