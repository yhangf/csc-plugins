# TR1 DFX 需求基线（DFX Baseline）

> **[AI 读取引导]**
> 本文件整合了 TR1 用户需求阶段所有 DFX 基线要求。
>
> **使用方式**：
> - 填写用户需求模板 §3.2 DFX 需求章节时，对照本文件中的 L0 底线要求逐项自检
> - AI 评审时，检查被评审文档是否覆盖了本文件中的 L0 必达项
> - L0 = 底线（必须达成）、L1 = 演进（按业务选择）、L2 = 拔高（追求卓越）

---

## 高密度摘要

**本文件的目标**：整合所有 DFX 基线文件的核心要求，作为 TR1 用户需求阶段 DFX 需求章节的填写依据和评审标准。

**全局强约束**：
- **必须**：所有产品必须满足 L0 底线要求，不满足项须有 Story 跟踪
- **必须**：新产品/新平台架构转标准化时，必须全量对照基线逐条检视
- **禁止**：仅填写"DFX 需求已考虑"等模糊声明，必须有 Checklist 自检结果

**基线等级定义**：

| 等级 | 含义 | 要求 |
|---|---|---|
| L0 | 底线 | 必须达成，不满足须有 Story 跟踪 |
| L1 | 演进 | 按业务需要选择性达成 |
| L2 | 拔高 | 追求卓越，可选 |

## 第一部分：性能基线

### RULE-DFX-PERF-001: Web 控制台页面响应性能（L0 底线）

- **等级**：L0（底线，必须达成）
- **适用范围**：所有具有 Web 控制台的产品
- **基线要求**：
  - 高频页面响应流畅（≤3 秒）占比：≥99%，不出现难以忍受（≥7 秒）的页面
  - 低频页面响应流畅（≤3 秒）占比：≥90%，难以忍受（≥7 秒）的页面：≤1%
  - 任何情况下不允许出现打开失败的页面
- **检查方式**：按产品实际高频/低频页面分类，逐页面测试响应时间

### RULE-DFX-PERF-002: 业务可用性指标（L0 底线）

- **等级**：L0（底线，必须达成）
- **适用范围**：所有产品
- **基线要求**：
  - 丝滑（99.99%~100%）占比：≥90%
  - 正常（99%~99.99%）占比：≤10%
  - 难以忍受（95%~99%）占比：0%
  - 业务响应时间 <1 秒占比：≥90%
- **检查方式**：在标准负载下测试业务可用性指标

### RULE-DFX-PERF-003: 共存影响指标（L0 底线）

- **等级**：L0（底线，必须达成）
- **适用范围**：与客户系统共存部署的产品
- **基线要求**：
  - 对客户现有业务系统的性能劣化 <5%，覆盖率 ≥90%
- **检查方式**：在客户典型业务负载下测试共存场景

## 第二部分：可靠性基线

### RULE-DFX-REL-001: 过载保障（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 过载检测阈值：CPU >80% 告警，>90% 熔断
  - 分级保护能力：L1（1 倍流量）~L4（10 倍流量）四级保护
  - 过载时业务降级而非崩溃

### RULE-DFX-REL-002: 系统高危操作限制（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 限制 root 直接访问，特权提升须有审计记录
  - 不自动删除用户对象，破坏性操作须二次确认
  - 命令行操作使用全路径命令

### RULE-DFX-REL-003: SLA 业务可用度（L0 底线）

- **等级**：L0（底线，必须达成），部分子项 L1~L4 分级
- **基线要求**：
  - 故障自动检测率：L1 ≥30% → L4 ≥100%
  - 故障自动检测时间：<5 分钟 → <10 秒（按等级）
  - 故障自动恢复率：L1 ≥30% → L4 ≥98%
  - 故障自动恢复时间：<10 分钟 → <30 秒（按等级）
  - 人工恢复时间：KA 客户 <10 分钟，普通客户 <1 小时
  - 业务中断时间：<0.5 小时 → 0（按等级）

### RULE-DFX-REL-004: 容灾 RTO/RPO（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 核心业务系统：RTO ≤2 分钟，RPO = 0
  - 运维管理系统：RTO ≤4 小时，RPO ≤1 小时
  - 遵循 SHARE-78 层级定义（Tier 0~6）
  - 来源参考：SHARE-78 行业标准（存储网络行业协会灾难恢复等级标准）

### RULE-DFX-REL-005: 集群扩容与高可用（L0/L1/L2 分级）

- **等级**：L0~L2 分级
- **基线要求**：
  - L0：支持自动扩缩容、零停机扩展、会话保持、负载均衡
  - L1：站点级故障转移、集群冗余、进程级恢复
  - L2：网络冗余、组件冗余、数据冗余（重要数据 5 个 9，关键数据 6 个 9）

## 第三部分：可服务性基线（可交付性 + 可维护性）

### RULE-DFX-SVC-001: 标准化部署（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - XaaS 类产品：部署时间 ≤3 小时（理想 1 小时内）
  - 网关类产品：部署时间 ≤0.5 天
  - 平台类产品：部署时间 ≤1~2 天
  - 终端类产品：部署时间 ≤1 天（MGR + 50 万支节点）

### RULE-DFX-SVC-002: 系统安装（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 单节点安装时间 ≤30 分钟
  - 支持一键部署
  - 安装后自动验证检查

### RULE-DFX-SVC-003: 系统升级（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 升级路径跳跃 ≤3 个版本
  - 大版本升级时间 ≤30 分钟，小版本 ≤20 分钟，补丁 ≤10 分钟
  - 升级前检查 ≤15 分钟
  - 支持回滚

### RULE-DFX-SVC-004: 初始化配置（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 单节点初始化 ≤10 分钟，集群 ≤20 分钟
  - 支持一键部署

### RULE-DFX-SVC-005: 授权管理（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 授权操作 ≤3 步完成
  - 重新授权不影响业务
  - IO 中断 ≤5 秒

### RULE-DFX-SVC-006: 系统监控与告警（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 监控粒度：30 分钟内 ≤10 秒，1 小时~1 天 ≤1 分钟，1 天~1 周 ≤1 小时
  - 告警抑制机制（防抖防洪）
  - 支持邮件/短信/IM 多渠道通知

### RULE-DFX-SVC-007: 故障处理与日常巡检（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 客户/渠道自助解决率 30%~50%
  - 支持网络诊断和故障隔离（bypass）
  - 巡检完成时间 ≤10 分钟
  - 支持巡检工具（aCheck/aDeploy/AIOPS）

### RULE-DFX-SVC-008: 日志管理（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 操作日志保留 ≥180 天
  - 安全日志保留 ≥1 年
  - 审计日志保留 ≥3 年
  - 来源参考：等保 2.0 合规要求

## 第四部分：开放性基线

### RULE-DFX-OPEN-001: API 标准化（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 对外 API 统一使用 OpenAPI 规范
  - 提供 Swagger 文档
  - API 版本向后兼容（L1）

### RULE-DFX-OPEN-002: 浏览器兼容性（L0 底线）

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - Chrome 80+
  - Edge 79+
  - Safari 13.1+
  - Firefox 72+

### RULE-DFX-OPEN-003: 日志外吐与 SIEM 集成（L1 演进）

- **等级**：L1（演进，按业务需要选择）
- **基线要求**：
  - 日志格式支持 CEF/LEEF/OCSF 标准
  - 支持 Syslog/Kafka 导出（L2）
  - 支持第三方设备日志接入（syslog/SNMP/WMI/JDBC）
  - 支持自定义解析和关联规则

### RULE-DFX-OPEN-004: 运维监控集成（L1 演进）

- **等级**：L1（演进，按业务需要选择）
- **基线要求**：
  - 支持 SNMP v1/v2c/v3
  - 支持 Zabbix/Prometheus + Grafana 集成
  - 告警格式支持 OpenTelemetry
  - 支持邮件/IM 告警

### RULE-DFX-OPEN-005: 认证集成（L1 演进）

- **等级**：L1（演进，按业务需要选择）
- **基线要求**：
  - 支持外部认证：LDAP/RADIUS/TACACS+/SSO/SAML/Kerberos
  - 支持 MFA：Google Authenticator/MS Authenticator
  - 支持 SCIM 2.0/OIDC 协议

### RULE-DFX-OPEN-006: 生态兼容性（L1 演进）

- **等级**：L1（演进，按业务需要选择）
- **基线要求**：
  - 国产 ARM/CPU 适配
  - 国产操作系统适配
  - 第三方备份软件支持（Veeam/Veritas）

## 第五部分：产品规格基线

### 规格基线质量属性分类

**功能性**：
- 通用功能
- 国际化

**非功能性（DFX）**：
- 性能、安全性、可靠性、易用性、合规性、兼容性、开放性、硬件、可交付性、可维护性、可行销性

### RULE-DFX-SPEC-001: 产品规格基线全量自检

- **等级**：L0（底线，必须达成）
- **基线要求**：
  - 对照产品规格基线覆盖维度表，逐维度填写本产品规格参数
  - 相对上版本有变化（升高或降低）的规格项须注明变化原因
  - 支持的硬件平台须明确列出，不支持的须说明原因
  - 授权模式设计须知会数字化平台授权中心

### 规格基线覆盖维度

产品规格基线覆盖约 800+ 条标准化条目（具体数量随版本迭代更新），覆盖以下维度：

| 维度 | 来源体系 | 对应基线 |
|---|---|---|
| 国际化 | 海外体系 | 海外基线 |
| 安全性 | 研发体系 | DFX 基线 |
| 开放性 | 研发体系 | DFX 基线 |
| 可交付性 | 技服体系 | DFX 基线 |
| 可靠性 | 研发体系 | DFX 基线 |
| 可维护性 | 技服体系 | DFX 基线 |
| 体验 | 研发体系 | DFX 基线 |
| XAAS 订阅 | 客户成功团队 | XAAS 基线 |
